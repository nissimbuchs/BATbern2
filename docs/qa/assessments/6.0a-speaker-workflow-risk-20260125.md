## Critical Risks Requiring Immediate Attention

### 1. API-001: No REST API Exposes Workflow Endpoints

**Score: 9 (Critical)**

**Probability**: High - Controller does not exist, verified via codebase search.

**Impact**: High - Without REST API, downstream stories (6.1b, 6.2, 6.3) cannot trigger workflow state changes. Frontend cannot update speaker states. Epic 6 is blocked.

**Affected Components**:
- `SpeakerWorkflowController` (missing)
- `speaker-coordination-api.openapi.yml`
- API Gateway routes

**Mitigation**:
| Strategy | Action | Owner | Timeline |
|----------|--------|-------|----------|
| Preventive | Create SpeakerWorkflowController with PUT endpoint | dev | Before Phase 3 |
| Preventive | Update OpenAPI spec with workflow endpoints | dev | Before Phase 3 |
| Preventive | Add API Gateway route configuration | dev | Before Phase 3 |
| Detective | Integration test for endpoint availability | dev | Phase 5 |

**Testing Requirements**:
- Test 3.1: `should_return200_when_validStateTransition`
- Test 3.2: `should_return404_when_speakerNotFound`
- Test 3.3: `should_return422_when_invalidTransition`
- Test 3.4: `should_return401_when_unauthorized`

**Residual Risk**: Low - Once controller implemented and tested
