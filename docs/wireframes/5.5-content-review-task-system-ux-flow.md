## Overview

This document describes the UX integration for three major feature areas:
1. **Speaker Content Submission** - How organizers submit speaker materials (AC6-10)
2. **Quality Review Queue** - How moderators review and approve content (AC11-15)

## Finalized UX Flow (Simplified)

Based on stakeholder feedback, the implementation uses a **streamlined integrated approach**:

1. **Content Submission & Quality Review**: Single drawer with context-aware tabs (Event Detail page)
2. **Task Dashboard**: Widget integrated into main Organizer Dashboard
3. **Event Task Configuration**: Tab in existing EventDetailEdit modal

- Less specialized review UI

## Recommended Approach: OPTION 1 (Integrated Drawer)

### Why Option 1?

1. **Natural Workflow Progression**
   - Speaker moves IDENTIFIED → CONTACTED → READY → ACCEPTED
   - Natural next step: Submit content (same page, same context)
   - No context switching required

2. **Leverages Existing Patterns**
   - Already using drawers for SpeakerOutreach
   - Users familiar with drawer interaction
   - Consistent UX across organizer features

3. **Minimal Navigation Overhead**
   - All speaker actions in one place
   - No need to remember separate pages
   - Reduces clicks from 5+ to 2

4. **Visual Progress Tracking**
   - Kanban lanes show content submission status
   - Easy to see which speakers need content
   - Status badges indicate completion

5. **Parallel Workflow Support**
   - Clear visual separation of SUB vs REV vs CNF lanes
   - Easy to understand parallel progression
   - Drag-drop still works for status changes

### Implementation Plan

**Phase 1: Extend Speaker Status Lanes**
- Add 3 new lanes: CONTENT_SUBMITTED, QUALITY_REVIEWED, CONFIRMED
- Update status color mapping
- Add visual badges for parallel workflow states

**Phase 2: Content Submission Drawer**
- Add "Content Submission" tab to existing speaker drawer
- Reuse UserLookupAutocomplete component (already exists)
- Integrate portrait upload with S3 presigned URLs
- Form validation with character counter

**Phase 3: Quality Review Queue (Separate Page)**
- Create `/organizer/content-review` route (Option 2 for moderators)
- Kanban board with review workflow
- Drawer with quality checks UI
- Approve/reject actions

**Phase 4: Task Dashboard**
- Create `/organizer/tasks` route
- Kanban board for task management
- Task completion modal
- Auto-create tasks on workflow transitions

## User Stories & Flows

### Flow 1: Organizer Submits Speaker Content (AC6-10)

**Actor:** Alice (Organizer)
**Precondition:** Speaker "Lisa Thompson" has accepted invitation (status=ACCEPTED)

1. Alice navigates to `/organizer/events/123`
2. Views Speaker Status Dashboard with Kanban lanes
3. Sees Lisa in "ACCEPTED" lane (green badge)
4. Clicks Lisa's card → Speaker Content Drawer opens
5. Switches to "Content Submission" tab
6. Searches "Lisa Thompson" in user autocomplete → found
7. Fills presentation title: "Building Scalable Microservices"
8. Fills abstract: "In this talk, I'll share lessons learned..." (145 chars)
9. Uploads portrait photo (optional)
10. Clicks [Submit Content]
11. Lisa's card animates to "CONTENT_SUBMITTED" lane (yellow badge)
12. Drawer closes, status summary updates: SUBMITTED count increases
13. System creates session record + session_users junction
14. SpeakerWorkflowStateChangeEvent published

**Result:** Lisa's status = CONTENT_SUBMITTED, session created, ready for review

---

### Flow 2: Moderator Reviews Content (AC11-15)

**Actor:** Bob (Moderator)
**Precondition:** Lisa's content submitted, status=CONTENT_SUBMITTED

1. Bob navigates to `/organizer/content-review`
2. Sees "Pending Review (5)" column with Lisa's card
3. Clicks [Review] on Lisa's card → Review Drawer opens
4. Reviews presentation details:
   - Title: "Building Scalable Microservices"
   - Abstract: 145 chars
5. Quality checks auto-run:
   - ✅ Length OK (145/1000)
   - ✅ "Lessons learned" detected
   - ✅ No product promotion
   - ✅ Professional tone
6. Bob approves → clicks [Approve]
7. Lisa's card moves to "Approved" column
8. System updates speaker_pool.status = QUALITY_REVIEWED
9. System checks: Does Lisa have slot assigned? (session.start_time != null)
10. If YES → auto-update to CONFIRMED + session_users.is_confirmed = true
11. If NO → stays in QUALITY_REVIEWED, waits for slot assignment

**Result:** Lisa's content approved, status = QUALITY_REVIEWED (or CONFIRMED if slot assigned)

---

### Flow 3: Organizer Creates Event with Tasks (AC19-27)

**Actor:** Carol (Organizer)
**Precondition:** Creating new event

1. Carol navigates to `/organizer/events/create`
2. Completes Steps 1-4 (Event Info, Venue, Team, Timeline)
3. Reaches Step 5: Task Configuration
4. Sees 7 default templates pre-checked:
   - ☑ Venue Booking (trigger: topic_selection, due: -90d)
   - ☑ Partner Meeting (trigger: topic_selection, due: 0d)
   - ☑ Moderator Assignment (trigger: topic_selection, due: -14d)
   - ☑ Newsletter: Topic (trigger: topic_selection, due: immediate)
   - ☑ Newsletter: Speakers (trigger: agenda_published, due: -30d)
   - ☑ Newsletter: Final (trigger: agenda_finalized, due: -14d)
   - ☑ Catering (trigger: agenda_finalized, due: -30d)
5. Assigns each task to team members via dropdowns
6. Clicks [+ Add Custom Task] → modal opens
7. Creates custom task:
   - Name: "Sponsor Outreach"
   - Trigger: Topic Selection
   - Due: 60 days before event
   - Assign: Carol
   - ☑ Save as template
8. Clicks [Add Task] → task added to list
9. Clicks [Next] → completes event creation
10. Event transitions to TOPIC_SELECTION state
11. EventTaskService listens to EventWorkflowTransitionEvent
12. Auto-creates 4 tasks (venue, partner, moderator, newsletter:topic)
13. Carol navigates to `/organizer/tasks`
14. Sees "To Do (4)" with newly created tasks

**Result:** Event created with 8 tasks, 4 auto-created and ready for assignment

## UX Decisions (Finalized 2025-12-20)

### 1. Content Review Access Control
**DECISION:** Any ORGANIZER can review content (not restricted to MODERATOR role)
- Faster reviews, distributed workload
- Team can self-organize review assignments

### 2. Speaker Card Visual Design in New Lanes
**DECISION:** Same card design, different header color by status
```
┌─────────────────┐
│ Lisa Thompson   │ ← Yellow background (SUBMITTED)
│ Tech AG         │
│ "Microservices" │ ← Show presentation title
│ Submitted: 2h   │ ← Relative timestamp
└─────────────────┘
```

### 3. Quality Review Rejection Flow
**DECISION:** Stay in CONTENT_SUBMITTED with "Changes Requested" visual indicator
- Speaker remains in CONTENT_SUBMITTED lane
- Red border + badge: "⚠️ Changes Requested"
- Organizer can see feedback in drawer
- Resubmit updates content, removes badge

### 4. Task Dashboard Location
**DECISION:** Integrate into main Organizer Dashboard (not separate page)
- Task widget on main dashboard showing critical/overdue tasks
- Quick view of all tasks across all events
- No dedicated `/organizer/tasks` page needed

### 5. Task Creation Timing
**DECISION:** Create all tasks immediately at event creation
- Tasks created with status="pending" until trigger state reached
- Then auto-transition to status="todo"
- Organizer sees full task list upfront for planning

### 6. Portrait Upload Location (Additional Decision)
**DECISION:** Remove portrait upload from Content Submission drawer
- Portrait upload already exists in UserCreateModal/UserEditModal
- Keep Content Submission focused on presentation details only
- If organizer needs to update portrait, use "Edit Speaker Profile" button

### 7. Quality Review UI Pattern (Simplified)
**DECISION:** No dedicated review page - use drawer from Event Detail
- Click speaker in CONTENT_SUBMITTED lane → Quality Review drawer opens
- Same drawer pattern as Content Submission, different tab
- Streamlined UX, no context switching

### 8. Event Task Configuration (Integration)
**DECISION:** Task configuration as tab in EventDetailEdit modal (not separate step)
- Existing EventDetailEdit has tabs: Info, Venue, Team, etc.
- Add new "Tasks" tab for task template selection
- Inline with existing modal pattern

## Finalized Navigation Map

```
/organizer/events (EventManagementDashboard)
    ├─ Right Sidebar: Task Widget (overdue + due soon)
    │   └─ [View All Tasks] → Task Board Modal (pending/todo/completed)
    │
    └─ Click Event Card
        ↓
    /organizer/events/:eventCode (EventDetail)
        ├─ Speaker Status Kanban Lanes (7 lanes: ID/CTD/RDY/ACC/SUB/REV/CNF)
        │   ├─ Click speaker in ACCEPTED → Content Submission Drawer
        │   │       └─ Submit → Speaker moves to SUB lane
        │   │
        │   └─ Click speaker in SUB/REV → Quality Review Drawer
        │           ├─ [Approve] → Speaker moves to REV (or CNF if slot assigned)
        │           └─ [Request Changes] → Feedback modal → stays in SUB with badge
        │
        └─ [Edit Event Details] button
            ↓
        EventDetailEdit Modal
            └─ Tabs: Info, Venue, Team, Timeline, [Tasks ✓], Actions
                └─ Tasks Tab: Select default + custom task templates
```
