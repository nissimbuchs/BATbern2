**I want** to upload session materials (presentations, PDFs, videos) directly through the session edit modal,

**Proposed Schema Extensions:**

```sql
-- New table: session_materials
CREATE TABLE session_materials (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    session_id UUID NOT NULL REFERENCES sessions(id) ON DELETE CASCADE,
    upload_id VARCHAR(100) UNIQUE NOT NULL,          -- From generic upload service
    s3_key VARCHAR(500) NOT NULL,                     -- S3 object key
    cloudfront_url VARCHAR(1000) NOT NULL,           -- CDN URL for access
    file_name VARCHAR(255) NOT NULL,                 -- Original file name
    file_extension VARCHAR(10) NOT NULL,             -- pptx, pdf, mp4, etc.
    file_size BIGINT NOT NULL,                       -- Size in bytes
    mime_type VARCHAR(100) NOT NULL,                 -- application/pdf, video/mp4, etc.
    material_type VARCHAR(50) NOT NULL,              -- PRESENTATION, DOCUMENT, VIDEO, OTHER
    uploaded_by VARCHAR(255) NOT NULL,               -- Username who uploaded
    created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,

    -- Content extraction tracking (for future RAG search - Story 5.x+1)
    content_extracted BOOLEAN NOT NULL DEFAULT FALSE,
    extraction_status VARCHAR(20) DEFAULT 'PENDING', -- PENDING, IN_PROGRESS, COMPLETED, FAILED, NOT_APPLICABLE

    INDEX idx_session_materials_session_id (session_id),
    INDEX idx_session_materials_upload_id (upload_id),
    INDEX idx_session_materials_extraction_status (extraction_status) -- For RAG pipeline queries
);

-- Extend sessions table with materials summary (for efficient querying)
ALTER TABLE sessions ADD COLUMN materials_count INTEGER NOT NULL DEFAULT 0;
ALTER TABLE sessions ADD COLUMN has_presentation BOOLEAN NOT NULL DEFAULT FALSE;
ALTER TABLE sessions ADD COLUMN materials_status VARCHAR(20);  -- NONE, PARTIAL, COMPLETE

-- Trigger to update sessions.materials_count automatically
CREATE OR REPLACE FUNCTION update_session_materials_count()
RETURNS TRIGGER AS $$
BEGIN
    UPDATE sessions SET
        materials_count = (SELECT COUNT(*) FROM session_materials WHERE session_id = NEW.session_id),
        has_presentation = (SELECT COUNT(*) > 0 FROM session_materials WHERE session_id = NEW.session_id AND material_type = 'PRESENTATION')
    WHERE id = NEW.session_id;
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER trigger_update_session_materials_count
AFTER INSERT OR DELETE ON session_materials
FOR EACH ROW EXECUTE FUNCTION update_session_materials_count();
```

## Acceptance Criteria

1. **AC1: Upload Materials in SessionEditModal**
   - SessionEditModal has tabbed interface with "Details" and "Materials" tabs
   - Materials tab shows FileUpload component with extended file type support (PPTX, PDF, MP4, MOV, AVI, DOCX, any file type)
   - Maximum file size: 100MB (increased from 5MB for videos)
   - Supports multiple file uploads (up to 10 files per session)
   - Each uploaded file shows: filename, file size, file type icon, delete button

2. **AC2: Materials Status in SpeakersSessionsTable**
   - Each session row shows materials status icon:
     - ❌ Missing (materials_count = 0)
     - ⚠️ Pending (materials_count > 0 && !has_presentation)
     - ✓ Complete (has_presentation)
   - Tooltip on icon shows: "X materials uploaded" or "No materials yet"
   - Clicking "Materials" button opens SessionEditModal on Materials tab

3. **AC3: Materials Metrics in EventOverviewTab**
   - Key Metrics card shows: "Materials: X/Y sessions complete"
   - X = sessions where has_presentation = true
   - Y = total sessions
   - Progress bar visualization (0-100%)
   - Color coding: Red (<50%), Yellow (50-99%), Green (100%)

4. **AC4: Materials Display in Public Archive**
   - EventProgram component shows materials section per session (only for archived events)
   - Materials grouped by type: Presentations, Documents, Videos
   - Each material shows: icon, filename, file size, download button
   - Download button opens CloudFront URL in new tab
   - Hidden for upcoming events (only shown when event is in past)

5. **AC5: Backend Materials CRUD**
   - POST `/sessions/{sessionSlug}/materials` - Associate uploaded materials with session
   - GET `/sessions/{sessionSlug}/materials` - List all materials for a session
   - DELETE `/sessions/{sessionSlug}/materials/{materialId}` - Remove a material
   - Materials are included in Session API responses (embedded list)

6. **AC6: File Type Support**
   - Allowed file types:
     - Presentations: `.pptx`, `.ppt`, `.key`, `.odp` (PRESENTATION type)
     - Documents: `.pdf`, `.doc`, `.docx`, `.txt` (DOCUMENT type)
     - Videos: `.mp4`, `.mov`, `.avi`, `.mkv`, `.webm` (VIDEO type)
     - Archives: `.zip`, `.tar.gz` (OTHER type)
   - File type validation on both frontend and backend
   - MIME type validation: Check file headers, not just extensions

7. **AC7: Speaker vs Organizer Permissions**
   - Speakers can upload/delete materials for their own sessions only
   - Organizers can upload/delete materials for any session
   - Backend enforces role-based access control (RBAC)
   - 403 Forbidden if speaker tries to modify another speaker's session

8. **AC8: Domain Event Emission**
   - Emit `SessionMaterialsUploadedEvent` when material is successfully associated
   - Event payload: `{ sessionSlug, eventCode, uploadId, materialType, uploadedBy }`
   - Used for audit trail and future analytics integrations
   - Material is created with `extraction_status = PENDING` (for future RAG pipeline - Story 5.x+1)

### RBAC Pattern (AC7) - CRITICAL FOR ALL TASKS

**Pattern Established in Tasks 3a & 3b** (applies to frontend and all future tasks):

**Backend Pattern** (matches ALL existing controllers):
```java
@PreAuthorize("hasRole('ORGANIZER') or hasRole('SPEAKER')")
public ResponseEntity<?> endpoint(...) {
    String username = securityContextHelper.getCurrentUsername();

    // AC7: Inline permission check - speakers can only modify own sessions
    if (!securityContextHelper.hasRole("ORGANIZER")) {
        Session session = sessionRepository.findBySessionSlug(sessionSlug)
                .orElseThrow(() -> new SessionNotFoundException(sessionSlug));

        if (!sessionUserRepository.existsBySessionIdAndUsername(session.getId(), username)) {
            throw new AccessDeniedException(
```

### Technical Design Notes

**File Upload Flow** (Reuse ADR-002):
```
1. User selects files in SessionEditModal Materials tab
2. Frontend calls POST /materials/presigned-url (generic endpoint, reuses logos service)
3. Backend returns presigned S3 URL + uploadId
4. Frontend uploads file directly to S3 (parallel uploads for multiple files)
5. Frontend calls POST /materials/{uploadId}/confirm for each file
6. User clicks Save → Frontend calls POST /sessions/{sessionSlug}/materials with uploadIds[]
7. Backend associates materials with session, copies S3 files to final location
8. Backend emits SessionMaterialsUploadedEvent
```

**S3 Key Strategy**:
```
Temporary: materials/temp/{uploadId}/file-{fileId}.{ext}
Final:     materials/{year}/events/{eventCode}/sessions/{sessionSlug}/file-{fileId}.{ext}

Example:
materials/2026/events/bat-bern-2026-spring/sessions/keynote-digital-transformation/file-f3e8d1a4.pptx
```

**Database Transaction Boundaries**:
- Material association: Single transaction (associate all materials atomically)
- Material deletion: Transaction includes S3 cleanup (compensating transaction if S3 fails)
- Trigger updates `sessions.materials_count` automatically (no explicit update needed)

**Content Extraction Tracking** (Forward Compatibility for Story 5.x+1):
- All materials created with `extraction_status = PENDING`, `content_extracted = false`
- No extraction processing in this story (placeholder for future RAG pipeline)
- Fields included in API responses but not used by frontend yet
- Future RAG pipeline will process PENDING materials and update status to COMPLETED

**Caching Strategy**:
- Session responses include embedded materials list (eager loading)
- CloudFront caches materials with 1-year TTL (immutable after upload)
- API responses cached in React Query with 5-minute stale time

**Error Handling Patterns**:
- S3 upload failure: Allow retry, cleanup temp files after 24 hours
- Material association failure: Rollback transaction, temp files remain for retry
- RBAC violation: Return 403 Forbidden with clear error message
- File type validation: Client-side + server-side (defense in depth)

**Performance Considerations**:
- Parallel S3 uploads for multiple files (use Promise.all)
- Lazy-load materials in SessionEditModal (fetch on tab switch)
- Paginate materials list if >20 files per session (unlikely)
- Use CloudFront for public downloads (reduce backend load)
