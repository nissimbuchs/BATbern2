## Story

**As a** speaker,
**I want** to access the speaker portal via a secure magic link in my invitation email,
**so that** I can respond to invitations and manage my content without creating a password.

## Architecture Context

### Architecture Patterns

**Referenced from architecture docs (SOURCE OF TRUTH):**

1. **Magic Link Security Model**:
   - JWT-based tokens with speaker claims
   - Token stored as SHA-256 hash in database
   - Single-use for state-changing actions (ACCEPT, DECLINE)
   - Reusable for read-only actions (VIEW dashboard)
   - Configurable expiry (default: 30 days)

2. **ADR-003** (Meaningful Identifiers):
   - Token contains `username`, `eventCode`, `sessionSlug`
   - No UUIDs exposed in magic link URLs

3. **ADR-004** (Cross-Service References):
   - Token validation returns username
   - User data fetched via HTTP enrichment

```sql
CREATE TABLE speaker_invitation_tokens (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    speaker_pool_id UUID NOT NULL REFERENCES speaker_pool(id) ON DELETE CASCADE,

CREATE INDEX idx_invitation_tokens_speaker ON speaker_invitation_tokens(speaker_pool_id);
CREATE INDEX idx_invitation_tokens_expires ON speaker_invitation_tokens(expires_at)
    WHERE used_at IS NULL;  -- Partial index for active tokens
```

## Acceptance Criteria

### AC1: Token Generation

1. `MagicLinkService.generateToken(speakerPoolId, action)` creates a secure token
2. Token is a 32-byte random value encoded as base64url (URL-safe)
3. Token hash (SHA-256) stored in database, plaintext never persisted
4. Token contains claims: `speakerPoolId`, `action`, `exp` (expiry)
5. Default expiry is 30 days from creation

### AC2: Token Validation

1. `MagicLinkService.validateToken(token)` returns `TokenValidationResult`
2. Validation checks:
   - Token hash exists in database
   - Token not expired (`expires_at > NOW()`)
   - Token not already used (for single-use actions)
3. Returns `TokenValidationResult` with: `valid`, `speakerPoolId`, `action`, `username`, `eventCode`
4. Invalid tokens return descriptive error: `EXPIRED`, `ALREADY_USED`, `NOT_FOUND`

### AC3: Single-Use Enforcement

1. Tokens with action `RESPOND` are single-use (accept/decline)
2. After successful use, `used_at` timestamp is set
3. Subsequent validation of used RESPOND token returns `ALREADY_USED` error
4. Tokens with action `VIEW` are reusable (dashboard access)
5. Tokens with action `SUBMIT` are reusable (content submission iterations)

### AC4: Token Expiry

1. Tokens expire after configurable duration (default: 30 days)
2. Expired tokens return `EXPIRED` error on validation
3. Scheduled job cleans up expired tokens older than 90 days
4. Token expiry is checked on every validation

### AC5: REST API Endpoint

1. `POST /api/v1/speaker-portal/validate-token` validates token and returns speaker context
2. Request body: `{ "token": "base64url-encoded-token" }`
3. Success response (200): `{ "valid": true, "username": "...", "eventCode": "...", "action": "..." }`
4. Error response (401): `{ "error": "TOKEN_EXPIRED" | "TOKEN_USED" | "TOKEN_NOT_FOUND" }`
5. Rate limited: 5 requests/minute per IP

### AC6: Security Requirements

1. Token is cryptographically random (SecureRandom, 32 bytes)
2. Only hash stored in database (SHA-256)
3. Token transmitted only via HTTPS
4. Token never appears in logs (masked in any logging)
5. Failed validation attempts logged with IP for audit

## Dev Notes - Implementation Guide

### Token Structure

```java
// Token generation
public String generateToken(UUID speakerPoolId, TokenAction action) {
    // 1. Generate cryptographically secure random bytes
    byte[] randomBytes = new byte[32];
    new SecureRandom().nextBytes(randomBytes);

    // 2. Encode as base64url (URL-safe)
    String token = Base64.getUrlEncoder().withoutPadding().encodeToString(randomBytes);

    // 3. Hash for storage
    String tokenHash = sha256(token);

    // 4. Store hash in database
    SpeakerInvitationToken entity = SpeakerInvitationToken.builder()
        .speakerPoolId(speakerPoolId)
        .tokenHash(tokenHash)
        .action(action)
        .expiresAt(Instant.now().plus(30, ChronoUnit.DAYS))
        .build();
    tokenRepository.save(entity);

    // 5. Return plaintext token (only time it exists in memory)
    return token;
}
```

### Magic Link URL Format

```
https://app.batbern.ch/speaker-portal/respond?token={base64url-token}

Examples:
- Respond to invitation: /speaker-portal/respond?token=abc123...
- Submit content: /speaker-portal/submit?token=xyz789...
- View dashboard: /speaker-portal/dashboard?token=def456...
```

### Token Actions

```java
public enum TokenAction {
    RESPOND,   // Accept/Decline invitation - SINGLE USE
    SUBMIT,    // Submit content - REUSABLE
    VIEW       // View dashboard - REUSABLE
}
```

### Validation Result

```java
public record TokenValidationResult(
    boolean valid,
    String error,           // null if valid, else: EXPIRED, ALREADY_USED, NOT_FOUND
    UUID speakerPoolId,
    String username,
    String eventCode,
    TokenAction action
) {
    public static TokenValidationResult valid(SpeakerInvitationToken token, SpeakerPool pool) {
        return new TokenValidationResult(true, null, pool.getId(),
            pool.getUsername(), pool.getEventCode(), token.getAction());
    }

    public static TokenValidationResult expired() {
        return new TokenValidationResult(false, "EXPIRED", null, null, null, null);
    }

    public static TokenValidationResult alreadyUsed() {
        return new TokenValidationResult(false, "ALREADY_USED", null, null, null, null);
    }

    public static TokenValidationResult notFound() {
        return new TokenValidationResult(false, "NOT_FOUND", null, null, null, null);
    }
}
```
