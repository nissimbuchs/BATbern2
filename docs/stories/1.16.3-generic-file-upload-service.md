# Story 1.16.3: Generic File Upload Service

## Status
Draft

## Story
**As a** developer,
**I want** a generic file upload service that is independent of entity creation,
**so that** files can be uploaded before entities exist, eliminating circular dependencies and enabling better user experience.

## Domain Context

### Problem Statement
Current architecture has circular dependency when creating companies with logos:
1. Logo upload endpoint requires company to exist: `POST /companies/{name}/logo/presigned-url`
2. Company cannot be created without being created first
3. Current workaround: Create company without logo, then edit to add logo (poor UX)
4. Same problem will affect user profiles, event banners, etc.

## Acceptance Criteria

1. **AC1: Generic Upload Initiation**
   - Generic presigned URL endpoint `POST /api/v1/logos/presigned-url` works without entity existence
   - Validates file type (PNG, JPEG, SVG) and size (max 5MB)
   - Creates Logo entity with status PENDING
   - Returns presigned S3 URL for temp location: `logos/temp/{uploadId}/`

2. **AC2: Upload Confirmation**
   - Confirmation endpoint `POST /api/v1/logos/{uploadId}/confirm` updates Logo status to CONFIRMED
   - Stores checksum for integrity verification
   - Logo record persists independently of any entity

3. **AC3: Entity Association**
   - Entity creation can reference uploaded file via logoUploadId field
   - Company creation with logoUploadId copies S3 file from temp to final location
   - Final S3 key format: `logos/{year}/{entity-type}/{entity-name}/logo-{fileId}.{ext}`
   - Logo status updated to ASSOCIATED with entity reference

4. **AC4: S3 File Management**
   - S3 copy operation from temp to final location on association
   - Original temp file cleaned up after successful copy
   - CloudFront URL generated for final location

5. **AC5: Automated Cleanup**
   - Scheduled job runs daily
   - Deletes PENDING logos older than 24 hours
   - Deletes CONFIRMED (not associated) logos older than 7 days
   - Corresponding S3 files also deleted

6. **AC6: Frontend Component Reusability**
   - FileUpload component supports generic mode (no entity) and entity-specific mode
   - useFileUpload hook handles presigned URL flow
   - Component location: `web-frontend/src/components/shared/FileUpload/` (NOT under Company)

7. **AC7: Legacy Endpoint Removal**
   - Old company-specific logo endpoints removed:
     - `POST /companies/{name}/logo/presigned-url` (DELETED)
     - `POST /companies/{name}/logo/confirm` (DELETED)
   - CompanyLogoService.java deleted entirely
   - Old LogoUpload.tsx component removed from Company folder

8. **AC8: Test Coverage**
   - Unit tests for GenericLogoService (upload, confirm, associate, copy)
   - Integration tests for LogoController endpoints
   - Unit tests for LogoCleanupService
   - Frontend tests for FileUpload component and useFileUpload hook
   - E2E test: Create company with logo in single operation

## Dev Notes - Implementation Guide

### Technical Design Notes

**State Machine:**
```
PENDING (upload requested)
   ↓
CONFIRMED (file uploaded to S3, checksum verified)
   ↓
ASSOCIATED (linked to entity, copied to final location)
```

**Service Architecture:**
- **GenericLogoService**: Core upload/confirm/associate logic
- **LogoCleanupService**: Scheduled job (@Scheduled cron)
- **LogoController**: REST API endpoints

**S3 Key Strategy:**
- Temp: `logos/temp/{uploadId}/logo-{fileId}.{ext}`
- Final: `logos/{year}/{entity-type}/{entity-name}/logo-{fileId}.{ext}`

**Transaction Boundaries:**
- Upload initiation: Single transaction (create Logo entity)
- Confirmation: Single transaction (update Logo status)
- Association: Transaction with company creation (atomic)

**Error Handling:**
- FileSizeExceededException (>5MB)
- InvalidFileTypeException (not PNG/JPEG/SVG)
- LogoNotFoundException (invalid uploadId)
- S3OperationException (copy/delete failures)

### API Contracts

**POST /api/v1/logos/presigned-url**
```yaml
requestBody:
  content:
    application/json:
      schema:
        type: object
        required: [fileName, fileSize, mimeType]
        properties:
          fileName:
            type: string
            example: "company-logo.png"
          fileSize:
            type: integer
            format: int64
            maximum: 5242880  # 5MB
            example: 102400
          mimeType:
            type: string
            enum: [image/png, image/jpeg, image/svg+xml]
            example: "image/png"

responses:
  '200':
    description: Presigned URL generated
    content:
      application/json:
        schema:
          type: object
          properties:
            uploadUrl:
              type: string
              format: uri
              example: "https://s3.amazonaws.com/bucket/key?signature=..."
            uploadId:
              type: string
              format: uuid
              example: "abc123-def456"
            fileId:
              type: string
              example: "f3e8d1a4"
            expiresInMinutes:
              type: integer
              example: 15
            requiredHeaders:
              type: object
              example: {"Content-Type": "image/png"}
```

**POST /api/v1/logos/{uploadId}/confirm**
```yaml
parameters:
  - name: uploadId
    in: path
    required: true
    schema:
      type: string

requestBody:
  content:
    application/json:
      schema:
        type: object
        required: [fileExtension]
        properties:
          fileExtension:
            type: string
            enum: [png, jpg, jpeg, svg]
            example: "png"
          checksum:
            type: string
            example: "sha256:abc123..."

responses:
  '200':
    description: Upload confirmed
    content:
      application/json:
        schema:
          type: object
          properties:
            uploadId:
              type: string
            status:
              type: string
              enum: [CONFIRMED]
```
