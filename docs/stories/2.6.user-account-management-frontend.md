# Story 2.6: User Account Management Frontend (Basic Profile + Settings)

## Status
Accepted

## Story

**As a** user (all roles: Organizer, Speaker, Partner, Attendee),
**I want** to view and manage my account through a unified interface with basic profile and settings,
**so that** I can maintain my personal information and configure essential preferences.

## Architecture Context

### Architecture Patterns
[Source: architecture/05-frontend-architecture.md]

**Component Architecture:**
- Atomic design with presentational/container separation
- Role-adaptive components that show/hide based on user roles
- Shared layout components from Story 1.17

**State Management:**
- Zustand for UI state (active tab, edit mode, dialog visibility)
- React Query for server state (user profile, preferences, settings, activity)
- Query caching: Profile (5min stale, 30min cache), Preferences/Settings (10min stale, 1hr cache)

**API Client Configuration:**
- Base URL includes `/api/v1` prefix from runtime config
- All service paths exclude `/api/v1` to avoid double concatenation
- Example: `const USER_API_PATH = '/users';` resolves to `http://localhost:8080/api/v1/users`

**Type Safety:**
- OpenAPI-generated TypeScript types from backend specifications
- Frontend-specific types for component props and UI state

**File Upload Pattern (ADR-002):**
- Three-phase upload: 1) Request presigned URL, 2) Upload to S3, 3) Confirm upload
- Association phase: Link uploaded file to user profile via `PATCH /users/me`
- S3 key structure: `logos/temp/{uploadId}/` → `logos/users/{userId}/profile-picture.*`

## Dev Notes - Implementation Guide

### Technical Design Notes

**Component Hierarchy:**
```
UserAccountPage (Page Container)
├── BaseLayout
│   ├── AppHeader (from Story 1.17)
│   └── Main Content
│       ├── Tabs (Profile, Settings)
│       ├── UserProfileTab
│       │   ├── ProfileHeader
│       │   │   └── ProfilePhotoUpload (ADR-002 modal)
│       │   ├── PersonalInformation
│       │   ├── RoleSpecificTabs (dynamic based on roles)
│       │   └── ActivityTimeline
│       └── UserSettingsTab
│           ├── Sub-Tabs (Account, Notifications, Privacy)
│           ├── AccountSettings
│           ├── NotificationSettings
│           └── PrivacySettings
```

**State Management Strategy:**

**Local Component State:**
- Active tab (Profile/Settings)
- Active settings sub-tab (Account/Notifications/Privacy)
- Edit mode boolean
- Photo upload dialog open/closed
- Form data during editing
- Form validation errors

**Zustand Global State:**
```typescript
// authStore (already exists from Story 1.17)
interface AuthStore {
  user: User | null;
  currentRole: UserRole;
  availableRoles: UserRole[];
  updateUser: (user: User) => void;
}
```

**React Query Server State:**
```typescript
// Query keys for caching
['user-profile', userId] // Stale: 5min, Cache: 30min
['user-preferences', userId] // Stale: 10min, Cache: 1hr
['user-settings', userId] // Stale: 10min, Cache: 1hr
['user-activity', userId, { limit: 5 }] // Stale: 1min, Cache: 10min
```

**Performance Optimizations:**
- Use React.memo() for expensive components (ActivityTimeline, RoleSpecificTabs)
- Lazy load photo upload dialog (React.lazy)
- Debounce bio character counter (300ms)
- Optimize re-renders with proper dependency arrays

**Error Handling:**
- Display correlation IDs in error messages for support debugging
- Show inline validation errors on form fields
- Display toast notifications for success/error operations
- Provide retry mechanisms for network failures
