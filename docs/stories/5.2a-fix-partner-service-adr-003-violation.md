# Story 5.2a: Fix ADR-003 Violation in Partner Topic Voting

## Status
**Priority**: CRITICAL - Prerequisite for Story 5.2
**Type**: Technical Debt / Bug Fix
**Estimated Effort**: 2-3 days

## Problem Statement

The current Partner Coordination Service implementation **violates ADR-003** by storing `topic_id` as a UUID instead of a meaningful identifier (String). This creates an architectural inconsistency and prevents proper implementation of Story 5.2 (Topic Selection System).

**Current (Incorrect):**
```java
// TopicVote.java line 53
@Column(name = "topic_id", nullable = false)
private UUID topicId;  // ❌ VIOLATES ADR-003
```

**Should be (per ADR-003 Rule 5):**
```java
@Column(name = "topic_id", nullable = false, length = 100)
private String topicId;  // ✅ Meaningful ID (e.g., "cloud-security-patterns")
```

## ADR-003 Requirement

Per [ADR-003: Meaningful Identifiers in Public APIs](../architecture/ADR-003-meaningful-identifiers-public-apis.md), lines 323-340:

> **CRITICAL: Services MUST store meaningful IDs for cross-service references, NOT UUIDs.**
>
> Cross-service references use meaningful IDs (companyName, username, eventCode, topicCode), NOT UUID foreign keys.

## User Story

**As a** system architect,
**I want** Partner Service to store topic IDs as meaningful identifiers (String),
**So that** the system complies with ADR-003 and enables proper cross-service integration without circular dependencies.

## Architecture Context

### Current State
- **Partner Service** stores votes with UUID topic references
- **Event Service** does not yet have Topics table (Story 5.2)
- **No data exists** in production (greenfield system)

### Target State
- **Partner Service** stores votes with String topic_code references (e.g., "cloud-security-patterns")
- **Event Service** can create topics with meaningful codes
- **No circular dependency**: Event Service → Partner Service (one-way)

## Technical Implementation

### 1. Database Migration

**File**: `services/partner-coordination-service/src/main/resources/db/migration/V3__fix_topic_id_to_string.sql`

```sql
-- Fix ADR-003 violation: Change topic_id from UUID to VARCHAR (meaningful ID)
-- Safe for greenfield: No production data exists

-- Step 1: Add new column with meaningful ID type
ALTER TABLE topic_votes ADD COLUMN topic_code VARCHAR(100);

-- Step 2: Data migration (none needed - greenfield)
-- If data existed: UPDATE topic_votes SET topic_code = ... WHERE topic_id IS NOT NULL;

-- Step 3: Drop old UUID column
ALTER TABLE topic_votes DROP COLUMN topic_id;

-- Step 4: Rename new column to topic_id
ALTER TABLE topic_votes RENAME COLUMN topic_code TO topic_id;

-- Step 5: Add NOT NULL constraint
ALTER TABLE topic_votes ALTER COLUMN topic_id SET NOT NULL;

-- Step 6: Update unique constraint to use new type
ALTER TABLE topic_votes DROP CONSTRAINT topic_votes_topic_id_partner_id_key;
ALTER TABLE topic_votes ADD CONSTRAINT topic_votes_topic_id_partner_id_key
    UNIQUE (topic_id, partner_id);

-- Step 7: Recreate index for performance
DROP INDEX IF EXISTS idx_topic_votes_topic_id;
CREATE INDEX idx_topic_votes_topic_id ON topic_votes(topic_id);

-- Verification query
-- SELECT column_name, data_type, character_maximum_length
-- FROM information_schema.columns
-- WHERE table_name = 'topic_votes' AND column_name = 'topic_id';
```

### 3. Service Layer Updates

**File**: `services/partner-coordination-service/src/main/java/ch/batbern/partners/service/TopicVotingService.java`

**Changes:**
- Update method signature: `castVote(String companyName, String topicId, Integer voteValue)`
- Add new method: `getVoteSummaryForTopic(String topicCode)`

```java
public VoteSummary getVoteSummaryForTopic(String topicCode) {
    List<TopicVote> votes = topicVoteRepository.findByTopicId(topicCode);

    int totalVotes = votes.size();
    int weightedScore = votes.stream()
        .mapToInt(v -> v.getVoteWeight() * v.getVoteValue())
        .sum();

    return VoteSummary.builder()
        .topicCode(topicCode)
        .totalVotes(totalVotes)
        .weightedScore(weightedScore)
        .build();
}
```
