## Architecture Context

### Architecture Patterns
- **Spring Batch Architecture**: Chunk-oriented processing with item readers, processors, writers
- **Parallel Processing**: Step execution with thread pools for independent entity types
- **ETL Pattern**: Extract (JSON → POJOs), Transform (apply mappings), Load (REST API + S3)
- **Idempotency**: Support re-running migration without duplicates (check existence before create)
- **Progress Tracking**: Spring Batch job repository tracks execution state, retries, and failures

**Reference**:
- Spring Batch documentation: Chunk-oriented processing, parallel steps, skip policies
- Mapping specifications: `docs/migration/schema-mapping-specification.md`
- File structure: `docs/migration/s3-structure.md`

### Database Schema Reference

**Source Schema (Legacy Angular):**
- Location: `apps/BATspa-old/src/api/`
- Files: `sessions.json`, `topics.json`, `companies.json`
- Assets: `apps/BATspa-old/src/assets/` (presentations, photos, logos)

**Target Schemas (PostgreSQL via REST APIs):**
- Company Management: Flyway migrations V3 (companies), V4 (user_profiles)
- Event Management: Flyway migrations V2 (events, sessions), V7 (session_users)
- Speaker Coordination: Flyway migrations for speakers

**Migration Metadata Schema (Migration Tool's own database):**
```sql
-- Migration progress tracking
CREATE TABLE migration_job_execution (
    job_execution_id BIGSERIAL PRIMARY KEY,
    job_name VARCHAR(100) NOT NULL,
    start_time TIMESTAMP NOT NULL,
    end_time TIMESTAMP,
    status VARCHAR(20) NOT NULL,  -- STARTED, COMPLETED, FAILED
    exit_message TEXT,
    records_processed INTEGER,
    records_failed INTEGER
);

-- Entity ID mapping (legacy ID → new UUID)
CREATE TABLE entity_id_mapping (
    id BIGSERIAL PRIMARY KEY,
    entity_type VARCHAR(50) NOT NULL,  -- Company, User, Event, Session, Speaker
    legacy_id VARCHAR(255) NOT NULL,
    new_id UUID NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE (entity_type, legacy_id)
);

-- Migration errors for manual review
CREATE TABLE migration_errors (
    id BIGSERIAL PRIMARY KEY,
    job_execution_id BIGINT REFERENCES migration_job_execution(job_execution_id),
    entity_type VARCHAR(50) NOT NULL,
    legacy_id VARCHAR(255),
    error_message TEXT NOT NULL,
    stack_trace TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## Acceptance Criteria

**Batch Processing Infrastructure (AC 1-4):**
1. **Spring Batch Setup**: Configure Spring Batch with job repository and task execution
2. **Batch Jobs**: Create jobs for each entity type (CompanyMigrationJob, EventMigrationJob, UserMigrationJob, SpeakerMigrationJob, SessionMigrationJob, FileMigrationJob)
3. **Chunk Processing**: Implement chunk-oriented processing (100 records per chunk)
4. **Sequential Execution**: Configure sequential execution with dependency order (Company → Event → User → Speaker → Session → File)

**Event Data Migration (AC 5-8):**
5. **Event Migration**: Migrate 60 historical events to Event Management Service using mappings from Story 3.1.2
6. **Session Migration**: Migrate 302 event sessions with speaker assignments
7. **Timeline Migration**: Create default timeline data for historical events (single workflow step: "archived")
8. **Metadata Migration**: Migrate event metadata (topics, descriptions, dates with German date parsing)

**Speaker Data Migration (AC 9-12):**
9. **Speaker Profile Migration**: Migrate 269 unique speaker profiles with validation (User + Speaker entities per ADR-004)
10. **Photo Migration**: Upload speaker photos to S3 with CDN URL generation (follow ProfilePictureService pattern)
11. **CV Migration**: Upload speaker CVs to S3 with access controls (if available in legacy data)
12. **Speaking History**: Establish SessionUser relationships (speaker-event-session assignments)

**Content & Media Migration (AC 13-16):**
13. **Presentation Files**: Migrate 258 presentations to S3 (PDF files) following S3 structure from Story 3.1.2
14. **Event Photos**: Migrate 578 event photo galleries to S3
15. **Thumbnail Generation**: Generate thumbnails for presentations (first page as PNG) and photos (300x300)
16. **CDN Configuration**: Update all file references to CloudFront CDN URLs (`https://cdn.batbern.ch/`)

**Company Data Migration (AC 17-20):**
17. **Company Profiles**: Create 65 company records (after deduplication) in Company Management Service
18. **Partner Status**: Set all migrated companies to `isVerified = false` (per Story 3.1.2 spec)
19. **Employee Relationships**: Establish User.companyId relationships using normalized company names (max 12 chars)
20. **Logo Migration**: Migrate 25 company logos (local files) + download 8 online logos to S3

### Technical Design Notes

**Spring Batch Architecture:**

1. **Job Configuration Pattern:**
```java
@Configuration
public class CompanyMigrationJob {

    @Bean
    public Job companyMigrationJob(JobBuilderFactory jobs, Step companyStep) {
        return jobs.get("companyMigrationJob")
            .incrementer(new RunIdIncrementer())
            .start(companyStep)
            .build();
    }

    @Bean
    public Step companyStep(StepBuilderFactory steps,
                            ItemReader<LegacyCompany> reader,
                            ItemProcessor<LegacyCompany, CompanyDto> processor,
                            ItemWriter<CompanyDto> writer) {
        return steps.get("companyStep")
            .<LegacyCompany, CompanyDto>chunk(100)  // Chunk size from AC3
            .reader(reader)
            .processor(processor)
            .writer(writer)
            .faultTolerant()
            .skipLimit(10)
```

**S3 Upload with Multipart:**

```java
@Service
public class S3UploadService {

    @Autowired
    private S3Client s3Client;

    @Value("${migration.s3.bucket-name}")
    private String bucketName;

    private static final long MULTIPART_THRESHOLD = 10 * 1024 * 1024;  // 10MB

    /**
     * Upload file to S3 with automatic multipart for large files
     * @return CloudFront CDN URL
     */
    public String uploadFile(File file, String s3Key) {
        try {
            if (file.length() > MULTIPART_THRESHOLD) {
                return uploadMultipart(file, s3Key);
            } else {
                return uploadSinglePart(file, s3Key);
            }
        } catch (Exception e) {
            throw new FileUploadException("Failed to upload " + file.getName(), e);
        }
    }

    private String uploadSinglePart(File file, String s3Key) {
        PutObjectRequest request = PutObjectRequest.builder()
            .bucket(bucketName)
            .key(s3Key)
            .contentType(getContentType(file))
            .build();

        s3Client.putObject(request, RequestBody.fromFile(file));

        return generateCdnUrl(s3Key);
    }

    private String uploadMultipart(File file, String s3Key) {
        // Multipart upload for files >10MB
        CreateMultipartUploadRequest createRequest = CreateMultipartUploadRequest.builder()
            .bucket(bucketName)
            .key(s3Key)
            .contentType(getContentType(file))
            .build();

        String uploadId = s3Client.createMultipartUpload(createRequest).uploadId();

        // Upload parts (5MB chunks)
        List<CompletedPart> completedParts = new ArrayList<>();
        long partSize = 5 * 1024 * 1024;  // 5MB
        long fileSize = file.length();
        int partNumber = 1;

        try (FileInputStream fis = new FileInputStream(file)) {
            for (long offset = 0; offset < fileSize; offset += partSize) {
                long currentPartSize = Math.min(partSize, fileSize - offset);

                UploadPartRequest partRequest = UploadPartRequest.builder()
                    .bucket(bucketName)
                    .key(s3Key)
                    .uploadId(uploadId)
                    .partNumber(partNumber)
                    .build();

                byte[] buffer = new byte[(int) currentPartSize];
                fis.read(buffer);

                UploadPartResponse partResponse = s3Client.uploadPart(
                    partRequest,
                    RequestBody.fromBytes(buffer)
                );

                completedParts.add(CompletedPart.builder()
                    .partNumber(partNumber)
                    .eTag(partResponse.eTag())
                    .build());

                partNumber++;
            }

            // Complete multipart upload
            CompleteMultipartUploadRequest completeRequest = CompleteMultipartUploadRequest.builder()
                .bucket(bucketName)
                .key(s3Key)
                .uploadId(uploadId)
                .multipartUpload(CompletedMultipartUpload.builder()
                    .parts(completedParts)
                    .build())
                .build();

            s3Client.completeMultipartUpload(completeRequest);

            return generateCdnUrl(s3Key);

        } catch (Exception e) {
            // Abort multipart upload on failure
            s3Client.abortMultipartUpload(AbortMultipartUploadRequest.builder()
                .bucket(bucketName)
                .key(s3Key)
                .uploadId(uploadId)
                .build());
            throw e;
        }
    }

    private String generateCdnUrl(String s3Key) {
        return "https://cdn.batbern.ch/" + s3Key;
    }
}
```
