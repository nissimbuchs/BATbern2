# Story 2.8.4: Partner Contact Management

## Status
Draft

## Story

**As an** organizer,
**I want** to manage partner contacts through the Contacts tab with add/remove functionality and role assignment,
**so that** I can maintain accurate contact information for each partner company with designated primary, billing, technical, and marketing contacts.

## Domain Context

### Primary Domain
Partner Analytics Domain - Partner contact management for organizer workflows

### Involved Services
- **Frontend**: React 19 application (TypeScript, Material-UI v5, Zustand, React Query)
- **Backend**: Partner Coordination Service (REST APIs from Story 2.7)
- **Backend**: User Management Service (for user autocomplete and data enrichment via HTTP)
- **Cache**: React Query for client-side caching (10min TTL for contacts)

**Shared-Kernel Foundation** (automatically included):
- ErrorResponse (standardized error format with correlationId)
- HTTP client utilities (JWT propagation, error handling)

**OpenAPI Code Generation**:
- Generate TypeScript types from `docs/api/partners-api.openapi.yml`

### Cross-Domain Dependencies
- **Depends on**: Story 2.8.2 (Partner Detail View) for Contacts tab container
- **Depends on**: Story 1.17 (React Frontend Foundation) for shared components
- **Depends on**: Story 2.7 (Partner Coordination Service Foundation) for Partner Contact APIs
- **Depends on**: Story 2.1b (User Management Service) for user autocomplete and enrichment
- **Integrates with**: API Gateway for authentication

## Requirements Context

### Related Functional Requirements
- **FR8**: Partner strategic input (contacts manage topic voting and suggestions)

- **NFR-Performance**: Contact list load <1s, add/remove operations <500ms

## Architecture Context

### Architecture Patterns
[Source: architecture/05-frontend-architecture.md]
- **State Management**: React Query for server state (contacts list, mutations)
- **Form Management**: React Hook Form with Zod validation
- **Validation**: Zod schemas with business rule validation (primary contact required)
- **API Path Pattern**: Use `/partners/{companyName}/contacts` (WITHOUT `/api/v1` prefix)

[Source: architecture/04-api-design.md#partners-api]
- **List Contacts**: GET /partners/{companyName}/contacts (returns enriched user data)
- **Add Contact**: POST /partners/{companyName}/contacts with AddPartnerContactRequest
- **Remove Contact**: DELETE /partners/{companyName}/contacts/{username}
- **HTTP Enrichment (ADR-004)**: Backend calls User Service to enrich contact data
- **Validation**: Backend enforces at least one PRIMARY contact

[Source: architecture/03-data-architecture.md#partner-contact]
- **PartnerContact Entity**: id (UUID), partnerId, username (meaningful ID - ADR-003), contactRole, isPrimary
- **Contact Roles**: PRIMARY, BILLING, TECHNICAL, MARKETING
- **Business Rule**: At least one PRIMARY contact required per partner
- **UNIQUE Constraint**: (partnerId, username) - prevent duplicate contacts

### Database Schema Reference
N/A - Frontend story, no database migrations required. Backend schema handled in Story 2.7.

### Infrastructure Components
- **Frontend**: Contacts tab integrated into Partner Detail view
- **CDN**: User profile pictures from CloudFront

## Acceptance Criteria

1. [ ] **Contacts Tab** displays list of partner contacts
   - Integrated into Partner Detail view (Story 2.8.2)
   - Tab label: "Contacts" with count badge (e.g., "Contacts (5)")
   - Contact list with enriched user data (name, email, role, profile picture)
   - Primary contact indicator (⭐ star icon/badge)
   - Role badges (PRIMARY, BILLING, TECHNICAL, MARKETING with color coding)
   - User profile pictures (from CloudFront CDN, fallback to initials avatar)
   - [+ Add Contact] button (top right)
   - Empty state: "No contacts yet. Add the first contact." with [+ Add Contact] button

2. [ ] **Contact Cards** display comprehensive contact information
   - User profile picture (avatar)
   - Full name (firstName + lastName from enriched user data)
   - Email (from enriched user data)
   - Contact role (badge with icon)
     - 👤 PRIMARY (primary color)
     - 💳 BILLING (blue)
     - 🔧 TECHNICAL (orange)
     - 📧 MARKETING (green)
   - Primary indicator (⭐ for isPrimary=true)
   - Actions: [Edit Role] [Remove] buttons
   - Tooltip on hover: "Primary contact for this partnership"

3. [ ] **Add Contact Modal** allows adding new contacts
   - Modal title: "Add Contact"
   - User autocomplete field (search by username or email)
     - Min 2 characters to trigger search
     - Display: Username, full name, email
     - Loading indicator while searching
     - Empty state: "No users found"
   - Contact role dropdown (PRIMARY, BILLING, TECHNICAL, MARKETING)
   - Primary contact toggle (switch)
     - Disabled if partner already has primary contact
     - Tooltip: "This partner already has a primary contact"
   - Form actions: [Cancel] [Add]
   - Validation:
     - User required
     - Role required
     - Cannot add duplicate contact (same partner + username)
   - Error handling: Display API errors (409 for duplicate, 404 for user not found)

4. [ ] **Add Contact Operation** creates new partner contact
   - Click [Add] → POST /api/v1/partners/{companyName}/contacts
   - Request body: username, contactRole, isPrimary
   - Success: Close modal, refresh contacts list, show success toast
   - Error handling:
     - 409 Conflict: "This user is already a contact for this partner"
     - 404 Not Found: "User not found"
     - 400 Validation: Display validation errors
   - Loading state: Disable form, show spinner on [Add] button
   - Optimistic update: Add contact to list immediately, revert on error

5. [ ] **Edit Contact Modal** allows modifying contact details (Epic 8 - defer if not in 2.7)
   - Modal title: "Edit Contact"
   - User display (read-only, show username and full name)
   - Contact role dropdown (editable)
   - Primary contact toggle (editable)
     - Cannot unset primary if only primary contact
     - Warning: "At least one primary contact is required"
   - Form actions: [Cancel] [Save]
   - Validation: Role required, primary validation
   - Update operation: PATCH /partners/{companyName}/contacts/{username}

6. [ ] **Remove Contact** deletes contact with confirmation
   - Click [Remove] → Confirmation dialog
     - Title: "Remove Contact?"
     - Message: "Are you sure you want to remove [Full Name] ([email]) from this partnership?"
     - Actions: [Cancel] [Remove]
   - Click [Remove] in dialog → DELETE /api/v1/partners/{companyName}/contacts/{username}
   - Success: Refresh contacts list, show success toast
   - Error handling:
     - 400 Bad Request: "Cannot remove last primary contact"
     - 404 Not Found: "Contact not found"
   - Business rule validation: Cannot remove if only primary contact
   - Optimistic update: Remove from list immediately, revert on error

7. [ ] **Primary Contact Validation** enforces business rules
   - At least one PRIMARY contact required per partner
   - Cannot remove last primary contact (API returns 400)
   - Cannot unset primary toggle if only primary contact
   - When adding new primary contact, automatically unset previous primary (backend handles)
   - Visual indicator: Warning message when attempting invalid operation

8. [ ] **User Autocomplete** provides user search
   - Search users by username or email (debounced 300ms)
   - Call User Management Service API: GET /users/search?q={query}
   - Display: Username, full name, email, profile picture
   - Loading indicator while searching
   - Empty state: "No users found matching '{query}'"
   - Error handling: Display error message if search fails
   - Keyboard navigation (Arrow keys, Enter to select)

9. [ ] **Contact List Loading States** provide feedback
   - Initial load: Full-page skeleton (3-5 contact card skeletons)
   - Refresh: Show loading indicator (do not hide existing contacts)
   - Empty state: "No contacts yet" message with illustration
   - Error state: Error message with [Retry] button

10. [ ] **Responsive Design** works on mobile, tablet, desktop
    - Mobile (<640px): Single column, stacked contact cards, fullscreen modals
    - Tablet (640-1024px): Two columns contact grid
    - Desktop (>1024px): Three columns contact grid

11. [ ] **Accessibility** meets WCAG 2.1 AA standards
    - Keyboard navigation (Tab, Enter, Escape)
    - Screen reader announcements (contacts added/removed, modal open)
    - ARIA labels for all buttons and form fields
    - Focus indicators visible
    - Color contrast for role badges ≥4.5:1

12. [ ] **Internationalization** supports German and English
    - All UI text translated (buttons, labels, messages)
    - Role labels translated (PRIMARY → Primär, BILLING → Abrechnung, etc.)
    - Confirmation messages translated

13. [ ] **Integration Tests** verify API integration
    - Test list contacts API call with HTTP enrichment
    - Test add contact API call
    - Test remove contact API call
    - Test duplicate contact error (409)
    - Test primary contact validation (400)
    - Test user autocomplete API call
    - Test error scenarios (404, 500, network errors)

**useAddPartnerContact.ts:**
```typescript
import { useMutation, useQueryClient } from '@tanstack/react-query';
import { addPartnerContact } from '@/services/api/partnerApi';
import { toast } from 'react-toastify';

export const useAddPartnerContact = (companyName: string) => {
  const queryClient = useQueryClient();

  return useMutation({
    mutationFn: (request: AddPartnerContactRequest) => addPartnerContact(companyName, request),
    onMutate: async (newContact) => {
      // Optimistic update
      await queryClient.cancelQueries({ queryKey: ['partner', companyName, 'contacts'] });
      const previousContacts = queryClient.getQueryData(['partner', companyName, 'contacts']);

      queryClient.setQueryData(['partner', companyName, 'contacts'], (old: any) => [
        ...old,
        { ...newContact, id: 'temp-id' },
      ]);

      return { previousContacts };
    },
    onSuccess: () => {
      queryClient.invalidateQueries({ queryKey: ['partner', companyName, 'contacts'] });
      toast.success('Contact added successfully');
    },
    onError: (error: any, newContact, context) => {
      // Revert optimistic update
      queryClient.setQueryData(['partner', companyName, 'contacts'], context?.previousContacts);

      const errorMessage = error.response?.data?.message || 'Failed to add contact';
      const correlationId = error.response?.data?.correlationId;
      toast.error(`${errorMessage}. Correlation ID: ${correlationId}`);
    },
  });
};
```
