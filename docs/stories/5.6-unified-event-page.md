# Story 5.6: Unified Event Page

## Status
In QA Review - Implementation Complete (pending E2E tests and deprecation cleanup)

## Story

**As an** organizer,
**I want** a single unified event page with tab-based navigation,
**so that** I can manage all aspects of an event (details, speakers, venue, team, publishing) without navigating between multiple confusing pages.

## Problem Statement

The current implementation has two separate pages that create confusion:
- **EventDetail.tsx** (`/organizer/events/:eventCode`) - Read-only view with SpeakerStatusDashboard
- **EventDetailEdit.tsx** (`/organizer/events/:eventCode/edit`) - "Edit" page that is mostly read-only with a modal for actual editing

**Issues:**
1. Both pages show similar read-only information despite naming
2. Speaker management is fragmented across 3 locations (EventDetail, EventDetailEdit, SpeakerOutreachPage)
3. Users are confused about which page to use
4. "Edit" button on EventDetail does nothing (just logs to console)
5. Navigation between pages required to see full event picture

### Acceptance Criteria

**AC1: Single Page with Tab Navigation**
- GIVEN an organizer accesses `/organizer/events/:eventCode`
- WHEN the page loads
- THEN they see a unified page with 6 tabs: Overview, Speakers, Venue & Logistics, Team, Publishing, Settings

**AC2: URL-Persisted Tab State**
- GIVEN an organizer is on the event page
- WHEN they click a tab (e.g., Speakers)
- THEN the URL updates to include `?tab=speakers`
- AND refreshing the page returns to the same tab

**AC3: Speakers Tab View Toggle**
- GIVEN an organizer is on the Speakers tab
- WHEN they use the view toggle
- THEN they can switch between Kanban, Table, and Sessions views
- AND the view is persisted in URL as `?tab=speakers&view=kanban`

**AC4: Kanban View with Drag-Drop**
- GIVEN an organizer is on the Speakers tab in Kanban view
- WHEN they drag a speaker card between status lanes
- THEN the speaker status is updated
- AND a confirmation dialog appears for status changes

**AC5: Table View with Bulk Actions**
- GIVEN an organizer is on the Speakers tab in Table view
- WHEN they select multiple speakers
- THEN they can perform bulk actions (Mark as Contacted, Assign Organizer, Send Reminder)

**AC6: Sessions View with Assignment**
- GIVEN an organizer is on the Speakers tab in Sessions view
- WHEN they click on an empty session slot
- THEN they see a dropdown to assign an accepted speaker from the pool

**AC7: Add Speakers Inline Panel**
- GIVEN an organizer clicks "Add Speakers" button
- WHEN the panel expands
- THEN they can quick-add a speaker or import from CSV
- AND the panel is collapsible

**AC8: Speaker Details Drawer**
- GIVEN an organizer clicks on a speaker card (any view)
- WHEN the drawer opens
- THEN they see speaker details, outreach history, and action buttons
- AND the drawer slides in from the right

**AC9: Deprecated Routes Redirect**
- GIVEN a user accesses `/organizer/events/:eventCode/edit`
- WHEN the page loads
- THEN they are redirected to `/organizer/events/:eventCode`
- AND the same applies for `/organizer/events/:eventCode/speakers/outreach` → `?tab=speakers`

**AC10: Mobile Responsive Design**
- GIVEN an organizer accesses the page on mobile
- WHEN the page loads
- THEN tabs become bottom navigation icons
- AND kanban lanes stack vertically as a grouped list

**AC11: Overview Tab Content**
- GIVEN an organizer is on the Overview tab
- WHEN they view the page
- THEN they see: status bar, theme image, event details, key metrics, quick actions

**AC12: Settings Tab with Danger Zone**
- GIVEN an organizer is on the Settings tab
- WHEN they scroll to Danger Zone
- THEN they see Cancel Event and Delete Event buttons with appropriate warnings

## Architecture Context

### UI Components

**New Components:**

1. **`EventPage.tsx`** (page container)
   - Props: None (uses route params)
   - Material-UI: `Tabs`, `Tab`, `Box`, `Container`
   - State: Active tab from URL query param
   - Hooks: `useSearchParams` for tab state, `useEvent` for data

2. **`EventOverviewTab.tsx`**
   - Props: `{ event: EventUI }`
   - Reuses: Status bar, theme image display, metrics cards
   - Material-UI: `Paper`, `Grid`, `Typography`, `Chip`

3. **`EventSpeakersTab.tsx`** (key consolidated component)
   - Props: `{ eventCode: string }`
   - State: View mode (kanban/table/sessions), add panel open
   - Reuses: `SpeakerStatusLanes`, `SpeakerOutreachDashboard`, `SpeakersSessionsTable`
   - Material-UI: `ToggleButtonGroup`, `Collapse`

4. **`EventVenueTab.tsx`**
   - Props: `{ event: EventUI }`
   - Reuses: `VenueLogistics` component
   - Additional: Schedule display

5. **`EventTeamTab.tsx`**
   - Props: `{ event: EventUI, eventCode: string }`
   - New: Team assignment table, outreach distribution stats

6. **`EventPublishingTab.tsx`**
   - Props: `{ event: EventUI }`
   - Displays: Publishing timeline, quality checkpoints, actions

7. **`EventSettingsTab.tsx`**
   - Props: `{ event: EventUI, eventCode: string }`
   - Contains: Notifications management, danger zone

**Modified Components:**

1. **`SpeakerStatusLanes.tsx`** - No changes, reused as-is
2. **`SpeakersSessionsTable.tsx`** - No changes, reused as-is
3. **`SpeakerOutreachDashboard.tsx`** - Minor: Remove breadcrumbs when used inside tab

### State Management

**URL State (Query Params):**
```typescript
interface EventPageParams {
  tab?: 'overview' | 'speakers' | 'venue' | 'team' | 'publishing' | 'settings';
  view?: 'kanban' | 'table' | 'sessions';  // Only for speakers tab
  speaker?: string;  // Speaker ID to open in drawer
}
```

**Local State:**
- `addSpeakersOpen: boolean` - Add speakers panel expanded
- `selectedSpeakers: string[]` - For bulk actions in table view
- `drawerOpen: boolean` - Speaker details drawer
- `currentSpeaker: SpeakerPoolEntry | null` - Selected speaker

**Server State (React Query):**
- Existing hooks: `useEvent`, `useSpeakerPool`, `useSpeakerStatusSummary`
- No new API calls required

### Routing Changes

**App.tsx Updates:**

```typescript
// Remove these routes:
// - /organizer/events/:eventCode (EventDetail)
// - /organizer/events/:eventCode/edit (EventDetailEdit)
// - /organizer/events/:eventCode/speakers/outreach (SpeakerOutreachPage)

// Add new unified route:
<Route
  path="/organizer/events/:eventCode"
  element={
    <ProtectedRoute>
      <AuthLayout>
        <EventPage />
      </AuthLayout>
    </ProtectedRoute>
  }
/>

// Add redirects for deprecated routes:
<Route
  path="/organizer/events/:eventCode/edit"
  element={<Navigate to={`/organizer/events/${eventCode}`} replace />}
/>
<Route
  path="/organizer/events/:eventCode/speakers/outreach"
  element={<Navigate to={`/organizer/events/${eventCode}?tab=speakers`} replace />}
/>
```
