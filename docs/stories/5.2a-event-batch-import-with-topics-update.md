# Event Batch Import with Topic Assignment and Update Mode

## Goal

Enhance the existing event batch import functionality to:

1. **Import topic categories** from the legacy `topics.json` file
2. **Assign topics to events** during import by mapping categories to topic entities
3. **Support update mode** to update existing events instead of creating duplicates
4. **Provide field-level control** via checkboxes to selectively update specific fields

## Problem Statement

Currently, when importing events that already exist:
- ❌ Events are skipped (duplicate detection)
- ❌ No way to update existing events with new data
- ❌ Topic categories are not imported or assigned
- ❌ No control over which fields should be updated

## Desired Behavior

### Import Flow with Topic Assignment

```
1. User uploads topics.json
2. System parses events and extracts categories
3. System shows preview with:
   - Event details
   - Topic category
   - Whether event exists in database
   - Checkboxes for field selection
4. User selects:
   - Import mode (Create new / Update existing)
   - Fields to update (if update mode)
5. System processes each event:
   a. Create/Update event based on mode
   b. Look up topic by category name
   c. If topic doesn't exist, create it
   d. Assign topic to event via POST /events/{eventCode}/topics
6. System shows results with topic assignment status
```

### Field Selection (Update Mode)

When "Update existing" mode is selected, show checkboxes:
- [ ] Title
- [ ] Description
- [ ] Topic/Category
- [ ] Date
- [ ] Venue
- [ ] Organizer

Only checked fields will be included in the PATCH request.

## Technical Architecture

### Data Flow

```
topics.json
    ↓
parseEventsJson() → validates category field
    ↓
createImportCandidates() → preserves category in topicCategory
    ↓
EventBatchImportModal → shows preview with mode/field selection
    ↓
useEventBatchImport.importEvents()
    ↓
For each candidate:
  1. Create/Update event (POST or PATCH)
  2. getOrCreateTopicByCategory(category) → topicId
  3. assignTopicToEvent(eventCode, topicId)
    ↓
Result with success/failure counts
```

### Topic Caching Strategy

To avoid redundant API calls, implement a topic cache:

```typescript
const topicCache = new Map<string, string>(); // category → topicId

async function getOrCreateTopicByCategory(category: string): Promise<string> {
  // Check cache first
  if (topicCache.has(category)) {
    return topicCache.get(category)!;
  }

  // Fetch from API
  const topics = await topicApiClient.listTopics(`{"category":"${category}"}`);

  if (topics.length > 0) {
    topicCache.set(category, topics[0].id);
    return topics[0].id;
  }

  // Create new topic
  const newTopic = await topicApiClient.createTopic({
    title: category,
    category: category,
    description: `Topic category: ${category}`
  });

  topicCache.set(category, newTopic.id);
  return newTopic.id;
}
```
