## Story

**As a** user who has received a password reset code,
**I want** to enter the code and set a new password in my preferred language,
**so that** I can regain access to my account securely.

**Context:** This story implements the Reset Password CONFIRMATION flow. This is PART 2 of the password reset flow. Users arrive here after requesting a reset in Story 1.2.2, receiving a 6-digit code via email from Cognito.

## Acceptance Criteria

### Reset Password Confirmation Screen
1. **Email Display**: Show email address (from URL param or state)
2. **Code Input Field**: 6-digit verification code input (text or numeric)
3. **New Password Field**: Password field with visibility toggle
4. **Confirm Password Field**: Password confirmation with visibility toggle
5. **Password Strength Indicator**: Real-time password strength feedback (weak/medium/strong)
6. **Password Requirements**: Display password requirements (8+ chars, uppercase, lowercase, number)
7. **Reset Password Button**: Primary action that triggers confirmation
8. **Back to Login Link**: Navigation to return to login screen

### Form Validation
9. **Code Validation**: Required, exactly 6 digits
10. **Password Validation**: Minimum 8 characters, uppercase, lowercase, number
11. **Password Match**: Confirm password must match new password
12. **Client-Side Validation**: Validate before submission
13. **Real-Time Feedback**: Show validation errors as user types

### AWS Amplify v6 Integration
14. **Confirm Reset Password API**: Call Amplify v6 `confirmResetPassword({ username, confirmationCode, newPassword })`
15. **Success Handling**: Redirect to login screen with success message
16. **Auto-Login Option**: Optionally sign in user automatically after successful reset

### Error Handling
17. **Invalid Code Error**: Display localized error for invalid/mismatched code
18. **Expired Code Error**: Display error and link back to forgot password screen
19. **Invalid Password Error**: Display error for password not meeting requirements
20. **Rate Limit Error**: Display error message when too many attempts
21. **Network Error**: Display error with retry option
22. **Generic Error**: Fallback error message for unexpected failures

## Dev Notes - Implementation Guide

### Frontend Amplify v6 Implementation

**`web-frontend/src/hooks/useResetPassword/useResetPassword.ts`:**
```typescript
/**
 * useResetPassword Hook - Amplify v6 Direct Integration
 * Story 1.2.2a: Reset Password Confirmation
 */

import { useMutation } from '@tanstack/react-query';
import { confirmResetPassword } from 'aws-amplify/auth';

interface ResetPasswordInput {
  email: string;
  code: string;
  newPassword: string;
}

interface UseResetPasswordOptions {
  onSuccess?: () => void;
  onError?: (error: ResetPasswordError) => void;
}

export interface ResetPasswordError extends Error {
  type: 'invalidCode' | 'expiredCode' | 'invalidPassword' | 'rateLimitExceeded' | 'unknownError';
  statusCode?: number;
}

export const useResetPassword = (options?: UseResetPasswordOptions) => {
  return useMutation<void, ResetPasswordError, ResetPasswordInput>({
    mutationFn: async ({ email, code, newPassword }: ResetPasswordInput) => {
      try {
        // Amplify v6: confirmResetPassword updates password with code
        await confirmResetPassword({
          username: email,
          confirmationCode: code,
          newPassword: newPassword,
        });

        console.log('[useResetPassword] Password reset successful');
      } catch (error: unknown) {
        console.error('[useResetPassword] Password reset failed:', error);

        const errorCode =
          (error as { code?: string; name?: string }).code ||
          (error as { code?: string; name?: string }).name ||
          'UNKNOWN_ERROR';

        // Map Cognito errors to app errors
        switch (errorCode) {
          case 'CodeMismatchException':
            throw Object.assign(new Error('Invalid verification code'), {
              type: 'invalidCode' as const,
              statusCode: 400,
            });
          case 'ExpiredCodeException':
            throw Object.assign(new Error('Verification code has expired'), {
              type: 'expiredCode' as const,
              statusCode: 400,
            });
          case 'InvalidPasswordException':
            throw Object.assign(new Error('Password does not meet requirements'), {
              type: 'invalidPassword' as const,
              statusCode: 400,
            });
          case 'LimitExceededException':
            throw Object.assign(new Error('Too many password reset attempts'), {
              type: 'rateLimitExceeded' as const,
              statusCode: 429,
            });
          default:
            throw Object.assign(
              new Error((error as { message?: string }).message || 'Password reset failed'),
              {
                type: 'unknownError' as const,
                statusCode: 500,
              }
            );
        }
      }
    },
    onSuccess: options?.onSuccess,
    onError: options?.onError,
  });
};
```

### Password Validation Schema

**Reuse from Story 1.2.3** (`web-frontend/src/schemas/registrationSchema.ts`):
```typescript
import { z } from 'zod';

export const createResetPasswordSchema = (t: TFunction) =>
  z.object({
    code: z
      .string()
      .length(6, t('reset.errors.invalidCode'))
      .regex(/^\d+$/, t('reset.errors.invalidCode')),
    newPassword: z
      .string()
      .min(8, t('reset.passwordRequirements'))
      .regex(/[A-Z]/, t('reset.passwordRequirements'))
      .regex(/[a-z]/, t('reset.passwordRequirements'))
      .regex(/[0-9]/, t('reset.passwordRequirements')),
    confirmPassword: z.string(),
  })
  .refine((data) => data.newPassword === data.confirmPassword, {
    message: t('reset.errors.passwordMismatch'),
    path: ['confirmPassword'],
  });
```
