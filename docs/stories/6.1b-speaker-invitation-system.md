## Story

**As an** organizer,
**I want** to invite speakers by entering their email address and have the system automatically send a personalized invitation email with a magic link,
**so that** I can efficiently onboard speakers without manual account creation or email composition.

## Domain Context

### Primary Domain

**Speaker Coordination Domain** - Automated invitation workflow

## Acceptance Criteria

### AC1: Invite Speaker by Email

1. `POST /api/v1/events/{eventCode}/speakers/invite` accepts email and optional name
2. Request body: `{ "email": "speaker@example.com", "name": "John Doe", "sessionId": "uuid" }`
3. If User with email exists, link SpeakerPool to existing username
4. If User doesn't exist, auto-create User with role=SPEAKER
5. Create SpeakerPool entry with workflow_state=IDENTIFIED
6. Return created SpeakerPool with username

### AC2: Auto-Create User

1. Username generated from email: `john.doe@example.com` → `john.doe`
2. If username taken, append number: `john.doe.2`, `john.doe.3`
3. User created with:
   - `email`: from request
   - `firstName`, `lastName`: parsed from name (or email prefix if no name)
   - `role`: SPEAKER
   - `status`: PENDING
4. User creation is idempotent (same email returns same user)

### AC3: Send Invitation Email

1. `POST /api/v1/events/{eventCode}/speakers/{username}/send-invitation` triggers email
2. Request body (optional): `{ "responseDeadline": "2026-03-01", "contentDeadline": "2026-04-01" }`
3. Email sent via AWS SES using `speaker-invitation` template
4. Email contains:
   - Personalized greeting (speaker name)
   - Event details (name, date, location)
   - Session details (title, description) if assigned
   - Magic link for response (from Story 6.1a)
   - Response deadline (from request or default: 30 days)
   - Organizer contact info
5. Workflow state transitions: IDENTIFIED → INVITED
6. SpeakerPool updated:
   - `invited_at` = now
   - `response_deadline` = provided or default
   - `content_deadline` = provided or null

### AC4: Email Template

1. HTML template with BATbern branding
2. Responsive design (mobile-friendly)
3. Clear call-to-action buttons: "Accept Invitation", "Decline"
4. Plain text fallback for email clients without HTML support
5. Template variables:
   - `{{speakerName}}` - Speaker's name
   - `{{eventName}}` - Event name
   - `{{eventDate}}` - Event date formatted
   - `{{sessionTitle}}` - Session title (if assigned)
   - `{{acceptLink}}` - Magic link for acceptance
   - `{{declineLink}}` - Magic link for decline
   - `{{responseDeadline}}` - Deadline for response
   - `{{organizerName}}` - Organizer contact name
   - `{{organizerEmail}}` - Organizer contact email

### AC5: Batch Invitation

1. `POST /api/v1/events/{eventCode}/speakers/invite-batch` accepts array of emails
2. Request body: `{ "invitations": [{ "email": "...", "name": "..." }, ...] }`
3. Process all invitations (create users, send emails)
4. Return summary: `{ "sent": 5, "failed": 1, "errors": [...] }`
5. Partial failures don't roll back successful invitations

### AC6: Domain Events

1. `SpeakerInvitationSentEvent` published after email sent
2. Event contains: speakerPoolId, username, email, eventCode, magicLinkToken (hashed), sentAt
3. Event logged for audit trail

### AC7: Idempotency

1. Resending invitation to same speaker generates new token
2. Previous token remains valid until expiry
3. Only one SpeakerPool entry per speaker per event
4. Duplicate invite request returns existing SpeakerPool (doesn't create new)

## Dev Notes - Implementation Guide

### Invite Speaker Flow

```java
@Service
@RequiredArgsConstructor
public class SpeakerInvitationService {

    private final UserAutoCreationService userAutoCreationService;
    private final SpeakerPoolRepository speakerPoolRepository;
    private final MagicLinkService magicLinkService;
    private final EmailService emailService;
    private final DomainEventPublisher eventPublisher;

    @Transactional
    public SpeakerPool inviteSpeaker(String eventCode, InviteSpeakerRequest request) {
        // 1. Find or create User
        User user = userAutoCreationService.findOrCreateByEmail(
            request.getEmail(),
            request.getName()
        );

        // 2. Check if already invited to this event
        Optional<SpeakerPool> existing = speakerPoolRepository
            .findByEventCodeAndUsername(eventCode, user.getUsername());
        if (existing.isPresent()) {
            return existing.get();  // Idempotent
        }

        // 3. Create SpeakerPool entry
        SpeakerPool speakerPool = SpeakerPool.builder()
            .eventId(getEventId(eventCode))
            .username(user.getUsername())
            .speakerName(user.getFullName())
            .workflowState(SpeakerWorkflowState.IDENTIFIED)
            .sessionId(request.getSessionId())
            .build();

        return speakerPoolRepository.save(speakerPool);
    }

    @Transactional
    public void sendInvitation(String eventCode, String username) {
        SpeakerPool speakerPool = speakerPoolRepository
            .findByEventCodeAndUsername(eventCode, username)
            .orElseThrow(() -> new SpeakerNotFoundException(username));

        // 1. Generate magic link tokens
        String acceptToken = magicLinkService.generateToken(
            speakerPool.getId(), TokenAction.RESPOND);
        String declineToken = magicLinkService.generateToken(
            speakerPool.getId(), TokenAction.RESPOND);

        // 2. Build and send email
        SpeakerInvitationEmail email = buildInvitationEmail(
            speakerPool, acceptToken, declineToken);
        emailService.sendHtmlEmail(email);

        // 3. Update workflow state
        speakerPool.setWorkflowState(SpeakerWorkflowState.INVITED);
        speakerPool.setInvitedAt(Instant.now());
        speakerPoolRepository.save(speakerPool);

        // 4. Publish domain event
        eventPublisher.publish(new SpeakerInvitationSentEvent(
            speakerPool.getId(),
            speakerPool.getUsername(),
            email.getRecipientEmail(),
            eventCode,
            Instant.now()
        ));
    }
}
```

### Username Generation

```java
@Service
public class UserAutoCreationService {

    public String generateUsername(String email) {
        // john.doe@example.com → john.doe
        String baseUsername = email.split("@")[0]
            .toLowerCase()
            .replaceAll("[^a-z0-9.]", ".");

        // Check for collisions
        String username = baseUsername;
        int suffix = 2;
        while (userRepository.existsByUsername(username)) {
            username = baseUsername + "." + suffix++;
        }

        return username;
    }
}
```
