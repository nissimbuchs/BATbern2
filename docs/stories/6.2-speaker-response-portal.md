### API Contracts (OpenAPI Excerpts)

**Extended RespondToInvitationRequest** (extends existing from Story 6.1):

```yaml
# Additions to docs/api/speaker-invitations-api.openapi.yml

components:
  schemas:
    RespondToInvitationRequest:
      type: object
      required:
        - responseType
      properties:
        responseType:
          type: string
          enum: [ACCEPTED, DECLINED, TENTATIVE]
        declineReason:
          type: string
          maxLength: 2000
        notes:
          type: string
          maxLength: 2000
        # NEW FIELDS for Story 6.2:
        preferences:
          $ref: '#/components/schemas/SpeakerResponsePreferences'

    SpeakerResponsePreferences:
      type: object
      description: Preferences captured when speaker accepts invitation
      properties:
        preferredTimeSlot:
          type: string
          enum: [MORNING, AFTERNOON, NO_PREFERENCE]
        travelRequirements:
          type: string
          enum: [LOCAL, ACCOMMODATION, VIRTUAL]
        technicalRequirements:
          type: array
          items:
            type: string
            enum: [MAC_ADAPTER, REMOTE_OPTION, SPECIAL_AV]
        initialPresentationTitle:
          type: string
          maxLength: 200
        commentsForOrganizer:
          type: string
          maxLength: 2000
```
