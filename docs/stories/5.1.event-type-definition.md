### Technical Design Notes

**Design Patterns**:
1. **Domain-Driven Design**: EventType as a value object within the Event aggregate
2. **Template Pattern**: Event types serve as templates for event creation
3. **Strategy Pattern**: Slot calculation strategies differ per event type
4. **Repository Pattern**: EventTypeRepository for data access abstraction
5. **Cache-Aside Pattern**: Caffeine cache with lazy loading for event types

**Service Method Signatures**:
```java
public interface EventTypeService {
    List<EventSlotConfiguration> getAllEventTypes();
    EventSlotConfiguration getEventType(EventType type);
    EventSlotConfiguration updateEventType(EventType type, EventSlotConfiguration config);
    void validateSlotCount(EventType type, int slotCount);
    List<EventSlot> generateSlotTemplate(EventType type);
}
```

**Database Transaction Boundaries**:
- Read operations (GET): @Transactional(readOnly = true)
- Write operations (PUT): @Transactional with cache invalidation
- Slot validation: No transaction (in-memory calculation)

**Caching Strategy**:
```java
@Cacheable(value = "eventTypes", key = "#type")
public EventSlotConfiguration getEventType(EventType type) { ... }

@CacheEvict(value = "eventTypes", key = "#type")
public EventSlotConfiguration updateEventType(EventType type, ...) { ... }
```

**Error Handling Patterns**:
- NotFoundException when event type not found
- ValidationException when slot count out of range
- BadRequestException when invalid event type string provided
- All exceptions extend from shared-kernel DomainException

**Performance Considerations**:

import { eventTypeService } from '@/services/eventTypeService';

  });

export function useEventType(type: EventType) {
  return useQuery({
    queryKey: ['eventTypes', type],
    queryFn: () => eventTypeService.getEventType(type),
    enabled: !!type,
  });

  const queryClient = useQueryClient();

  return useMutation({

    onSuccess: () => {
      queryClient.invalidateQueries({ queryKey: ['eventTypes'] });
    },
  });
