### Technical Design Notes

**Design Patterns**:
1. **Domain-Driven Design**: EventType as a value object within the Event aggregate
2. **Template Pattern**: Event types serve as templates for event creation
3. **Strategy Pattern**: Slot calculation strategies differ per event type
4. **Repository Pattern**: EventTypeRepository for data access abstraction
5. **Cache-Aside Pattern**: Caffeine cache with lazy loading for event types

**Service Method Signatures**:
```java
public interface EventTypeService {
    List<EventSlotConfiguration> getAllEventTypes();
    EventSlotConfiguration getEventType(EventType type);
    EventSlotConfiguration updateEventType(EventType type, EventSlotConfiguration config);
    void validateSlotCount(EventType type, int slotCount);
    List<EventSlot> generateSlotTemplate(EventType type);
}
```

**Database Transaction Boundaries**:
- Read operations (GET): @Transactional(readOnly = true)
- Write operations (PUT): @Transactional with cache invalidation
- Slot validation: No transaction (in-memory calculation)

**Caching Strategy**:
```java
@Cacheable(value = "eventTypes", key = "#type")
public EventSlotConfiguration getEventType(EventType type) { ... }

@CacheEvict(value = "eventTypes", key = "#type")
public EventSlotConfiguration updateEventType(EventType type, ...) { ... }
```

**Error Handling Patterns**:
- NotFoundException when event type not found
- ValidationException when slot count out of range
- BadRequestException when invalid event type string provided
- All exceptions extend from shared-kernel DomainException

**Performance Considerations**:
- Event types are read-heavy (rarely updated) → aggressive caching
- Cache in-memory with Caffeine (avoid network latency to Redis)
- Slot template generation memoized per event type
- Index on event_types(type) for fast lookups

**Complete CREATE TABLE** [Source: docs/architecture/03-data-architecture.md]:

```sql
-- Event types configuration table
CREATE TABLE event_types (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    type VARCHAR(20) NOT NULL UNIQUE,  -- 'full_day', 'afternoon', 'evening'
    min_slots INTEGER NOT NULL CHECK (min_slots > 0),
    max_slots INTEGER NOT NULL CHECK (max_slots >= min_slots),
    slot_duration INTEGER NOT NULL CHECK (slot_duration >= 15),  -- minutes
    theoretical_slots_am BOOLEAN NOT NULL DEFAULT true,
    break_slots INTEGER NOT NULL DEFAULT 0,
    lunch_slots INTEGER NOT NULL DEFAULT 0,
    default_capacity INTEGER NOT NULL CHECK (default_capacity > 0),
    typical_start_time TIME,
    typical_end_time TIME,
    created_at TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT CURRENT_TIMESTAMP
);

-- Index for fast lookups by type
CREATE INDEX idx_event_types_type ON event_types(type);

-- Trigger to update updated_at timestamp
CREATE TRIGGER update_event_types_updated_at
    BEFORE UPDATE ON event_types
    FOR EACH ROW
    EXECUTE FUNCTION update_updated_at_column();
```
