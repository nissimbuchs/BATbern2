# Story 2.5.2: Event Management Frontend

**Linear Issue**: [BAT-5](https://linear.app/batbern/issue/BAT-5/event-management-frontend)

## Status
In progress

## Story

**As a** user (Organizer),
**I want** a React frontend interface for event management with dashboard, CRUD forms, workflow visualization, and timeline views,
**so that** I can efficiently manage events through an intuitive user interface.

## Domain Context

### Primary Domain
Event Management Domain - Frontend React application consuming Event Management Service APIs

### Involved Services
- **Frontend**: React application (TypeScript, Material-UI, Zustand, React Query)
- **Backend**: Event Management Service (REST APIs from Story 2.2)
- **Backend**: Speaker Coordination Service (speaker assignment data)
- **Backend**: Venue Management (venue booking and logistics)
- **Backend**: Company Management Service (company data for speakers)
- **Backend**: User Management Service (team assignments, organizer roles)
- **CDN**: CloudFront for event images and materials
- **Cache**: React Query for client-side caching (5-15min TTL)

## Architecture Context

### Architecture Patterns
[Source: architecture/05-frontend-architecture.md]
- **Component Architecture**: Atomic design with presentational/container separation
- **State Management**:
  - Zustand for UI state (filters, modals, view mode, selected event)
  - React Query for server state (events, workflow, speakers, cache management)
- **Routing**: React Router v6 with lazy loading
- **Form Management**: React Hook Form with Zod validation
- **Data Visualization**: Recharts for workflow progress, analytics charts

[Source: architecture/04-api-event-management.md]
- **API Client**: Axios with interceptors for auth/error handling
- **Resource Expansion**: Uses `?include=workflow,speakers,sessions,venue,registrations` to reduce API calls (90% reduction)
- **Filtering**: JSON filter syntax from Story 1.15a.1
- **Caching**: React Query with 5-15min staleTime for event data based on volatility

[Source: architecture/03-data-architecture.md#event-data-model]
- **Event Entity**: UUID identifier, title, description, eventDate, eventType, status, workflowState
- **TypeScript Types**: Generated from OpenAPI specifications

### Infrastructure Components
[Source: architecture/02-infrastructure-deployment.md]

**Frontend Deployment:**
- **S3 + CloudFront**: Static React app hosting
- **Environment Config**: Runtime environment variables via window.__ENV__
- **API Base URL**: Configured per environment (local, staging, production)

**CDN Integration:**
- **Event Images**: Served via CloudFront with 1-year max-age
- **Assets**: Vite build with content hashing for cache busting

## Acceptance Criteria

1. **Event Dashboard Display** (Main Screen)
   - Display active events pipeline with progress bars (65%, 15%, etc.)
   - Show workflow step (Step 7/16, Step 2/16) for each event
   - Display critical tasks count with priority indicators (⚠️, 🔴)
   - Implement team activity feed with real-time updates (manual reload)
   - Quick actions sidebar ([+ New Event], [📊 Analytics], [👥 Speakers], etc.)

2. **Event List & Filters**
   - Filter events by status (active, published, completed, archived)
   - Filter by year and event type
   - Search events by title
   - Sort by date, status, workflow progress
   - URL persistence for filters (shareable links)

3. **Create Event Form**
   - Modal form with event fields (title, description, date, type, theme, capacity)
   - Event type selection (Full Day, Afternoon, Evening)
   - Date picker with validation (min 30 days in future)
   - Registration deadline picker (min 7 days before event)
   - Venue selection dropdown
   - "Save Draft" button (allows incomplete data)
   - "Save & Create" button (full validation)

4. **Edit Event Form**
   - Pre-fill form with existing event data
   - Inline editing for title, description, theme
   - Auto-save enabled (5-second debounce) - NOT configurable
   - Partial update support (only changed fields sent to backend)
   - Unsaved changes warning on modal close
   - Role-based access (Organizers only)

5. **Workflow Progress Display**
   - Progress bar showing completion percentage
   - Current step indicator (Step X/16: Step Name)
   - Clickable progress bar navigates to Workflow Visualization
   - Warning indicators for blockers (⚠️)
   - [View Workflow Details] button

6. **Venue & Logistics Management**
   - Venue selection dropdown with details (address, capacity, amenities)
   - Booking status display (✓ Confirmed, Pending, Not Booked)
   - [Change Venue] button opens venue selector
   - Catering configuration section
   - [Configure Menu] button opens catering form

7. **Topic Management**
   - Display assigned topics (3 topics with cards)
   - Show topic usage history (Last used: BATbern #48)
   - Partner votes count display
   - [View] button shows topic details modal
   - [Remove] button unassigns topic
   - [+ Add Topic from Backlog] navigates to topic backlog

8. **Speakers & Sessions Display**
   - Table/grid showing speakers and sessions (12 slots)
   - Slot time display (09:00-10:00, 10:15-11:15, etc.)
   - Speaker name and company
   - Session title
   - Materials status (Complete ✓, Pending ⚠️, Missing ❌)
   - [View Details], [Edit Slot], [Materials] action buttons
   - [View Full Agenda] button
   - [Manage Speaker Assignments] button
   - [Auto-Assign Speakers] AI algorithm button

9. **Team Assignments**
   - Lead Organizer display (read-only)
   - Co-Organizers list
   - Moderator dropdown assignment
   - Content Reviewer dropdown assignment
   - [Manage Team Assignments] button

10. **Publishing Configuration**
    - Publishing strategy display (Progressive, Immediate)
    - Current phase indicator
    - Publishing timeline with checkpoints (✓ completed, ○ pending)
    - Quality checkpoints list
    - [Configure Publishing], [Republish Event], [Preview Public Page] buttons

11. **Notifications & Reminders**
    - Active automations count
    - Notification rules list with status
    - [Manage Notifications], [Send Manual Notification] buttons

12. **Critical Tasks**
    - Display prioritized tasks (overdue materials, venue confirmation, abstracts moderation)
    - Urgency indicators (⚠️ warning, 🔴 critical)
    - Inline action buttons ([Contact], [Extend Deadline], [Confirm], [Assign to Team])
    - Task completion updates task count

13. **Team Activity Feed**
    - Display team actions with timestamps
    - User attribution (Mark assigned, Anna completed, System auto-sent)
    - Activity type icons
    - Filter by user, event, action type
    - Manual reload to see new activity

14. **Performance**
    - Initial page load <2s (First Contentful Paint)
    - Event detail with all includes <500ms (Caffeine-cached <50ms)
    - Form submission feedback <200ms
    - Skeleton loading states for all data fetching
    - React Query caching reduces redundant API calls (90% reduction via `?include=`)

15. **Responsive Design**
    - Mobile-first design with breakpoints (xs: 0px, sm: 600px, md: 900px, lg: 1200px, xl: 1536px)
    - Collapsible sections on mobile (workflow, venue, topics, speakers)
    - Stacked layout for forms on mobile/tablet
    - Touch-friendly buttons (min 44x44px)
    - Bottom sheet for action menus on mobile

16. **Accessibility (WCAG 2.1 AA)**
    - All interactive elements keyboard accessible (Tab navigation)
    - ARIA labels for all buttons and inputs
    - Focus indicators (2px solid border)
    - Screen reader support (form labels, error announcements)
    - Color contrast ratio 4.5:1 minimum
    - Status indicators use both color and icon

17. **Error Handling**
    - Display user-friendly error messages for all API failures
    - Retry buttons for failed requests
    - Offline mode with cached data display
    - Validation errors shown inline with red text
    - Network error banner at top of screen
    - Concurrent edit conflict dialog (version mismatch detection)

18. **State Management**
    - Zustand for UI state (filters persist across navigation)
    - React Query for server state (automatic refetch on window focus)
    - Optimistic updates for mutations (immediate UI feedback)
    - Cache invalidation on event create/update/delete
    - Filter/sort state persists in URL query parameters

19. **Integration with Services**
    - Call Event Management Service for CRUD operations
    - Call Speaker Coordination Service for speaker details
    - Call Venue Management for venue selection
    - Call User Management for team assignments
    - Use consolidated APIs with `?include=workflow,speakers,sessions,venue` to reduce API calls
    - Handle 401 (unauthorized), 403 (forbidden), 404 (not found), 500 (server error)

20. **Auto-Save Functionality**
    - Auto-save always enabled for all organizers (5-second debounce)
    - NOT configurable per user (stakeholder decision from wireframe v1.1)
    - Visual indicator showing "Saving..." and "Last saved at [timestamp]"
    - Error banner if auto-save fails
    - Auto-save conflict detection (concurrent edit warning)

21. **Delete Event**
    - [Delete Event] button with organizer-only access
    - Deletion impact dialog shows counts (registrations, speakers, materials)
    - Validation: Cannot delete if registrations exist (must cancel event instead)
    - Organizer confirmation required (no admin approval needed per wireframe v1.1)
    - On success: Navigate back to dashboard with success message
    - On validation failure: Show "Cannot delete - cancel event instead" error

22. **Internationalization (i18n) - German/English** (Foundation from Story 1.17)
    - All UI labels, buttons, headings, and help text fully translated using react-i18next
    - Translation namespace: `events` (separate from `common`, `auth`, `validation`)
    - Translation files: `public/locales/de/events.json` and `public/locales/en/events.json`
    - Language switcher in AppHeader from Story 1.17 changes all event UI text
    - Type-safe translation keys with TypeScript autocomplete (generated from JSON files)
    - Date/time formatting with locale support (date-fns with de/enUS locales)
    - Number formatting with Swiss locale (CHF currency, 1'500.50 format)
    - Error messages translated (validation errors, API errors, confirmation dialogs)
    - No hardcoded strings in any component - all text via `t('events.key')`
    - Language preference persists via Story 1.17's language switcher (LocalStorage + User Profile API)

23. **Navigation Integration** (Foundation from Story 1.17)
    - Event Management menu items added to NavigationMenu from Story 1.17
    - Navigation items: "Events", "Create Event", "Event Timeline" (child items)
    - Menu items visible only for Organizer role (role-adaptive from Story 1.17)
    - Active route highlighting when on event pages
    - Navigation labels fully translated (German/English)
    - Breadcrumbs show current event context (e.g., "Events > BATbern56 > Edit")
    - Mobile hamburger menu includes event navigation (responsive from Story 1.17)

24. **Error Handling Integration** (Foundation from Story 1.9)
    - API errors display correlation ID from `X-Correlation-ID` response header
    - Error response format: `{ timestamp, path, status, error, message, correlationId }`
    - Error codes mapped to user-friendly messages:
      - ERR_VALIDATION → "Invalid input. Please check your data."
      - ERR_NOT_FOUND → "Event not found."
      - ERR_UNAUTHORIZED → "You don't have permission to access this event."
      - ERR_SERVICE → "An unexpected error occurred. Please try again."
    - Global error toast from Story 1.17 shows translated error messages
    - ErrorBoundary from Story 1.17 catches component-level errors
    - Retry buttons for failed API requests
    - Correlation ID included in error logs for debugging

export const useUpdateEvent = () => {
  const queryClient = useQueryClient();

  return useMutation({
    mutationFn: ({ eventCode, data }: { eventCode: string; data: UpdateEventRequest }) =>
      eventApiClient.updateEvent(eventCode, data),
    onMutate: async ({ eventCode, data }) => {
      // Optimistic update
      await queryClient.cancelQueries(['event', eventCode]);
      const previousEvent = queryClient.getQueryData(['event', eventCode]);
      queryClient.setQueryData(['event', eventCode], (old: Event) => ({ ...old, ...data }));
      return { previousEvent };
    },
    onSuccess: (_, { eventCode }) => {
      queryClient.invalidateQueries(['events']);
      queryClient.invalidateQueries(['event', eventCode]);
      toast.success('Event updated successfully');
    },
    onError: (error, { eventCode }, context) => {
      queryClient.setQueryData(['event', eventCode], context.previousEvent);
      if (error.response?.status === 409) {
        toast.error('Concurrent edit detected. Please reload and try again.');
      } else {
        toast.error('Failed to update event');
      }
    },
  });
};

import axios, { AxiosError } from 'axios';
import { Event, EventDetail, CreateEventRequest, UpdateEventRequest, EventFilters, EventWorkflow } from '../types/event.types';

const apiClient = axios.create({
  baseURL: import.meta.env.VITE_API_BASE_URL,
  timeout: 10000,
});

// Request interceptor for auth token
apiClient.interceptors.request.use((config) => {
  const token = localStorage.getItem('authToken');
  if (token) {
    config.headers.Authorization = `Bearer ${token}`;
  }
  return config;
});

// Response interceptor for error handling
apiClient.interceptors.response.use(
  (response) => response,
  (error: AxiosError) => {
    if (error.response?.status === 401) {
      // Redirect to login
      window.location.href = '/login';
    }
    return Promise.reject(error);
  }
);

export const eventApiClient = {
  getEvents: async (filters: EventFilters, page: number, limit: number): Promise<{ data: Event[]; pagination: any }> => {
    const params = new URLSearchParams();
    params.append('page', String(page));
    params.append('limit', String(limit));

    if (filters.status) {
      params.append('status', filters.status);
    }
    if (filters.year) {
      params.append('year', String(filters.year));
    }
    if (filters.eventType) {
      params.append('eventType', filters.eventType);
    }
    if (filters.searchQuery) {
      params.append('search', filters.searchQuery);
    }

    const response = await apiClient.get(`/api/v1/events?${params.toString()}`);
    return response.data;
  },

  getEvent: async (eventCode: string, include?: string[]): Promise<EventDetail> => {
    const params = include?.length ? `?include=${include.join(',')}` : '';
    const response = await apiClient.get(`/api/v1/events/${eventCode}${params}`);
    return response.data;
  },

  createEvent: async (data: CreateEventRequest): Promise<Event> => {
    const response = await apiClient.post('/api/v1/events', data);
    return response.data;
  },

  updateEvent: async (eventCode: string, data: UpdateEventRequest): Promise<Event> => {
