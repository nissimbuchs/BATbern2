# Story 2.8.3: Partner Create/Edit Modal

## Status
Ready for Review

## Story

**As an** organizer,
**I want** a Create/Edit Partner modal with company selection, tier management, and date configuration,
**so that** I can create new partnerships and modify existing partnership details through a user-friendly form interface.

## Domain Context

### Primary Domain
Partner Analytics Domain - Partner CRUD interface for organizer workflows

### Involved Services
- **Frontend**: React 19 application (TypeScript, Material-UI v5, Zustand, React Hook Form, Zod)
- **Backend**: Partner Coordination Service (REST APIs from Story 2.7)
- **Backend**: Company Management Service (for company autocomplete)
- **Cache**: React Query for client-side caching

**Shared-Kernel Foundation** (automatically included):
- ErrorResponse (standardized error format)
- Validation utilities

**OpenAPI Code Generation**:
- Generate TypeScript types from `docs/api/partners-api.openapi.yml`

### Cross-Domain Dependencies
- **Depends on**: Story 2.8.1 (Partner Directory) for [+ Add Partner] button trigger
- **Depends on**: Story 2.8.2 (Partner Detail View) for [Edit Partner] button trigger
- **Depends on**: Story 1.17 (React Frontend Foundation) for modal components
- **Depends on**: Story 2.7 (Partner Coordination Service Foundation) for Partner CRUD APIs
- **Depends on**: Story 2.1 (Company Management Service) for company autocomplete
- **Integrates with**: API Gateway for authentication

## Requirements Context

### Related Functional Requirements
- **FR8**: Partner strategic input (partnership creation/management)
- **FR12**: Partner meeting coordination (partnership tier affects benefits)
- **NFR-Performance**: Modal open <200ms, form submission <2s

[Source: architecture/05-frontend-architecture.md]

- **Modal Pattern**: Material-UI Dialog with fullscreen on mobile

## Acceptance Criteria

1. [ ] **Create Partner Modal** opens from Directory [+ Add Partner] button
   - Modal title: "Create Partnership"
   - Company autocomplete field (search companies by name)
   - Partnership tier dropdown (BRONZE, SILVER, GOLD, PLATINUM, STRATEGIC with emoji icons)
   - Partnership start date picker (default: today)
   - Partnership end date picker (optional)
   - Tier benefits preview (updates when tier changes)
   - [Cancel] and [Save] buttons
   - Validation: Company required, tier required, start date required
   - Error display: Inline validation errors, API errors with correlationId

2. [ ] **Edit Partner Modal** opens from Detail [Edit Partner] button
   - Modal title: "Edit Partnership"
   - Company display (read-only, show company name and logo)
   - Partnership tier dropdown (pre-filled with current tier)
   - Partnership start date picker (pre-filled with current date)
   - Partnership end date picker (pre-filled if exists, optional)
   - Tier benefits preview
   - [Cancel] and [Save] buttons
   - Validation: Same as create
   - Dirty detection: Only enable [Save] if form changed

3. [ ] **Company Autocomplete** provides company search
   - Search companies by name (debounced 300ms)
   - Display: Company name, logo, industry
   - Min 2 characters to trigger search
   - Loading indicator while searching
   - Empty state: "No companies found"
   - Error handling: Display error message if search fails

4. [ ] **Partnership Tier Dropdown** displays all tiers with visual indicators
   - Options: 🏆 Strategic, 💎 Platinum, 🥇 Gold, 🥈 Silver, 🥉 Bronze
   - Tooltip with tier description on hover
   - Icon + label display in dropdown
   - Selected tier highlights in primary color

5. [ ] **Partnership Date Pickers** handle date selection
   - Start date: Required, default today, cannot be future date
   - End date: Optional, must be after start date
   - Date format: DD.MM.YYYY (German) or MM/DD/YYYY (English)
   - Calendar popup with MUI X DatePicker
   - Keyboard input support
   - Validation: End date > start date

6. [ ] **Tier Benefits Preview** updates dynamically
   - Display benefits list based on selected tier
   - Benefits update immediately when tier changes
   - Checkmark (✓) for included benefits
   - Benefits examples:
     - Strategic: Logo, Newsletter, Priority access, Meetings, Analytics, Dedicated manager
     - Platinum: Logo, Newsletter, Priority access, Meetings
     - Gold: Logo, Newsletter, Priority access
     - Silver: Newsletter, Event access
     - Bronze: Event access

7. [ ] **Form Validation** provides inline feedback
   - Required field validation (company, tier, start date)
   - Date range validation (end date > start date)
   - Error messages display below fields in red
   - Validation triggers: onBlur, onSubmit
   - i18n error messages (German/English)

8. [ ] **Create Partnership** saves new partnership
   - Click [Save] → POST /api/v1/partners
   - Request body: companyName, partnershipLevel, partnershipStartDate, partnershipEndDate (optional)
   - Success: Close modal, show success toast, invalidate partners cache, navigate to detail view
   - Error handling: Display API error with correlationId
   - Loading state: Disable form, show spinner on [Save] button

9. [ ] **Edit Partnership** updates existing partnership
   - Click [Save] → PATCH /api/v1/partners/{companyName}
   - Request body: partnershipLevel, partnershipStartDate, partnershipEndDate (optional)
   - Success: Close modal, show success toast, invalidate partner cache
   - Error handling: Display API error
   - Optimistic update: UI updates before API response

10. [ ] **Modal UX** provides smooth interactions
    - Modal opens with fade-in animation (200ms)
    - Modal closes on [Cancel], [X], or Escape key
    - Unsaved changes warning: Confirm before close if form is dirty
    - Fullscreen on mobile (<640px)
    - Centered dialog on desktop
    - Focus trap: Tab/Shift+Tab within modal
    - Autofocus on first field when modal opens

11. [ ] **Accessibility** meets WCAG 2.1 AA standards
    - Keyboard navigation (Tab, Shift+Tab, Enter, Escape)
    - Screen reader announcements (modal open, errors, success)
    - ARIA labels for all form fields
    - Focus indicators visible
    - Error messages linked to fields with aria-describedby

12. [ ] **Internationalization** supports German and English
    - All labels and messages translated
    - Date format respects locale
    - Tier labels translated
    - Error messages in correct language

13. [ ] **Integration Tests** verify API integration
    - Test create partner API call
    - Test update partner API call
    - Test company autocomplete API call
    - Test validation errors (400)
    - Test conflict errors (409 - partnership already exists)
    - Test network errors

### Zod Validation Schema

```typescript
// src/schemas/partnerSchema.ts
import { z } from 'zod';

export const CreatePartnerSchema = z.object({
  companyName: z.string().min(1, 'Company is required').max(12),
  partnershipLevel: z.enum(['BRONZE', 'SILVER', 'GOLD', 'PLATINUM', 'STRATEGIC'], {
    errorMap: () => ({ message: 'Partnership tier is required' }),
  }),
  partnershipStartDate: z.date({ required_error: 'Start date is required' }),
  partnershipEndDate: z.date().optional(),
}).refine(
  (data) => !data.partnershipEndDate || data.partnershipEndDate > data.partnershipStartDate,
  { message: 'End date must be after start date', path: ['partnershipEndDate'] }
);

export const UpdatePartnerSchema = z.object({
  partnershipLevel: z.enum(['BRONZE', 'SILVER', 'GOLD', 'PLATINUM', 'STRATEGIC']),
  partnershipStartDate: z.date(),
  partnershipEndDate: z.date().optional(),
}).refine(
  (data) => !data.partnershipEndDate || data.partnershipEndDate > data.partnershipStartDate,
  { message: 'End date must be after start date', path: ['partnershipEndDate'] }
);

export type CreatePartnerFormData = z.infer<typeof CreatePartnerSchema>;
export type UpdatePartnerFormData = z.infer<typeof UpdatePartnerSchema>;
```

### React Hook Form Integration

```typescript
// PartnerCreateEditModal.tsx
import { useForm } from 'react-hook-form';
import { zodResolver } from '@hookform/resolvers/zod';
import { CreatePartnerSchema, UpdatePartnerSchema } from '@/schemas/partnerSchema';

const { control, handleSubmit, formState: { errors, isDirty } } = useForm({
  resolver: zodResolver(mode === 'create' ? CreatePartnerSchema : UpdatePartnerSchema),
  defaultValues: mode === 'edit' ? {
    partnershipLevel: partnerToEdit.partnershipLevel,
    partnershipStartDate: new Date(partnerToEdit.partnershipStartDate),
    partnershipEndDate: partnerToEdit.partnershipEndDate ? new Date(partnerToEdit.partnershipEndDate) : undefined,
  } : {
    companyName: '',
    partnershipLevel: 'BRONZE',
    partnershipStartDate: new Date(),
    partnershipEndDate: undefined,
  },
});
```

### React Query Mutations

```typescript
// useCreatePartner.ts
import { useMutation, useQueryClient } from '@tanstack/react-query';
import { useNavigate } from 'react-router-dom';
import { createPartner } from '@/services/api/partnerApi';
import { toast } from 'react-toastify';

export const useCreatePartner = () => {
  const queryClient = useQueryClient();
  const navigate = useNavigate();

  return useMutation({
    mutationFn: createPartner,
    onSuccess: (data) => {
      queryClient.invalidateQueries({ queryKey: ['partners'] });
      queryClient.invalidateQueries({ queryKey: ['partner-statistics'] });
      toast.success('Partnership created successfully');
      navigate(`/partners/${data.companyName}`);
    },
    onError: (error: any) => {
      const correlationId = error.response?.data?.correlationId;
      toast.error(`Failed to create partnership. Correlation ID: ${correlationId}`);
    },
  });
};
```
