# Story 2.5.3a: Event Theme Image Upload

## Status
Not started

## Story

**As an** event organizer,
**I want** to upload and display a theme image for each event,
**so that** events have visual appeal and are more recognizable on the dashboard and detail pages.

## Architecture Context

### Architecture Patterns

**File Upload Pattern (ADR-002):**
```
Phase 1: Upload Initiation
Frontend → POST /logos/presigned-url → GenericLogoService
         ← {presignedUrl, uploadId, fileId}

Phase 2: S3 Upload & Confirmation
Frontend → PUT {presignedUrl} (binary) → S3
Frontend → POST /logos/{uploadId}/confirm → GenericLogoService

Phase 3: Entity Association (during event create/update)
Frontend → POST /events {title, ..., themeImageUploadId} → EventService
EventService → associateLogoWithEntity() → GenericLogoService
             → Copy S3: logos/temp/{uploadId}/ → logos/{year}/events/{eventCode}/
             → Update Logo status: CONFIRMED → ASSOCIATED
```

**S3 Key Strategy:**
- **Temporary**: `logos/temp/{uploadId}/logo-{fileId}.{ext}` (during upload)
- **Final**: `logos/{year}/events/{eventCode}/theme-{uploadId}.{ext}` (after association)

**Image Specifications:**
- **Max File Size**: 5 MB
- **Allowed Types**: image/png, image/jpeg, image/svg+xml
- **Recommended Dimensions**: 1920x400px (16:4.8 aspect ratio)
- **CDN Delivery**: Via CloudFront with 1-year cache

## Acceptance Criteria

### AC1: Backend - Event Entity Extension
**Given** the Event entity exists
**When** extending it with theme image fields
**Then**
- Event entity has `themeImageUrl` field (String, nullable)
- Event entity has `themeImageUploadId` field (String, nullable)
- Database migration V7 adds columns to events table
- OpenAPI spec updated with new fields

### AC2: Backend - Event Creation with Theme Image
**Given** a user has uploaded an image (Phase 1-2 complete)
**When** creating an event with `CreateEventRequest { ..., themeImageUploadId }`
**Then**
- EventService calls `genericLogoService.associateLogoWithEntity(uploadId, "EVENT", eventCode, finalS3Key)`
- S3 file copied from temp to final location: `logos/{year}/events/{eventCode}/theme-{uploadId}.png`
- Event.themeImageUrl set to CloudFront URL
- Logo status updated to ASSOCIATED
- Event returned with themeImageUrl in response

### AC3: Backend - Event Update with Theme Image
**Given** an existing event
**When** updating with PATCH /events/{eventCode} { themeImageUploadId }
**Then**
- New theme image associated (if uploadId provided)
- Old theme image remains (no automatic deletion - audit trail)
- Event.themeImageUrl updated to new CloudFront URL
- Logo status updated to ASSOCIATED

### AC4: Frontend - EventCard Theme Image Display
**Given** an event with themeImageUrl
**When** displaying EventCard on dashboard
**Then**
- Theme image displayed at top of card (full width, 200px height)

- Image loads with lazy loading
- Image has alt text: "Theme image for {eventTitle}"

#### Task 5: Implement Theme Image Association in Create
**File**: `services/event-management-service/src/main/java/ch/batbern/eventmanagement/service/EventService.java`

**Implementation**:
```java
public EventResponse createEvent(CreateEventRequest request) {
    // Create event entity
    Event event = Event.builder()
        .eventNumber(request.getEventNumber())
        .title(request.getTitle())
        .description(request.getDescription())
        .date(request.getDate())
        // ... other fields
        .themeImageUploadId(request.getThemeImageUploadId()) // Store uploadId
        .build();

    // Generate eventCode
    String eventCode = generateEventCode(event.getEventNumber());
    event.setEventCode(eventCode);

    // Save event (needed for eventCode)
    Event savedEvent = eventRepository.save(event);

    // Associate theme image if provided
    if (request.getThemeImageUploadId() != null) {
        associateThemeImage(savedEvent, request.getThemeImageUploadId());
    }

    return mapToResponse(savedEvent);
}

private void associateThemeImage(Event event, String uploadId) {
    try {
        // Generate final S3 key
        int year = LocalDate.now().getYear();
        String finalS3Key = String.format(
            "logos/%d/events/%s/theme-%s.png",
            year,
            event.getEventCode(),
            uploadId
        );

        // Call GenericLogoService to associate
        String cloudFrontUrl = genericLogoService.associateLogoWithEntity(
            uploadId,
            "EVENT",
            event.getEventCode(),
            finalS3Key
        );

        // Update event with CloudFront URL
        event.setThemeImageUrl(cloudFrontUrl);
        eventRepository.save(event);

        log.info("Theme image associated for event {}: {}",
                event.getEventCode(), cloudFrontUrl);

    } catch (Exception e) {
        log.error("Failed to associate theme image for event {}: {}",
                event.getEventCode(), e.getMessage());
        // Don't fail event creation - theme image is optional
    }
}
```

- Create event with themeImageUploadId associates image

public EventResponse updateEvent(String eventCode, UpdateEventRequest request) {
    Event event = findEventByCode(eventCode);

    // Update fields
    event.setTitle(request.getTitle());
    event.setDescription(request.getDescription());
    // ... other fields

    // Update theme image if new uploadId provided

    // Update theme image if provided
    if (request.getThemeImageUploadId() != null) {
        associateThemeImage(event, request.getThemeImageUploadId());
    }

    Event patchedEvent = eventRepository.save(event);
    return mapToResponse(patchedEvent);
}

## Notes

### Architecture Decision
This story follows ADR-002 (Generic File Upload Service) three-phase pattern:
1. Frontend uploads to S3 via presigned URL (no entity required)
2. Frontend confirms upload
3. Backend associates logo with event during create/update

### Key Benefits
- **Reuses existing infrastructure**: GenericLogoService handles all file operations
- **No entity coupling**: Theme images can be uploaded before event exists
- **Automatic cleanup**: Orphaned uploads removed after 24h (PENDING) or 7d (CONFIRMED)
- **CDN delivery**: CloudFront serves images with 1-year cache
- **Error resilience**: Event creation succeeds even if image association fails

### Image Specifications
- **Max Size**: 5 MB
- **Formats**: PNG, JPEG, SVG
- **Recommended Dimensions**: 1920x400px (16:4.8 aspect ratio)
- **Display**: 200px height on cards, 400px on detail page (desktop)

### S3 Key Pattern
- **Temporary**: `logos/temp/{uploadId}/logo-{fileId}.{ext}`
- **Final**: `logos/{year}/events/{eventCode}/theme-{uploadId}.{ext}`

### Error Handling
- Theme image upload is **optional** - event creation succeeds even if upload fails
- Broken image URLs fall back to gradient background
- User-friendly error messages for upload failures

### Future Enhancements
- Image optimization (auto-resize, WebP conversion)
- Multiple event images (gallery)
- Image cropping tool in upload component
- Background image variants for different themes
