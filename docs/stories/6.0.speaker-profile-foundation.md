## Architecture Context

### Architecture Patterns

**Referenced from architecture docs (SOURCE OF TRUTH):**

1. **ADR-003** (Meaningful Identifiers in Public APIs):
   - Speaker references User via `username` (NOT UUID)
   - API uses `username` as public identifier
   - Example: `GET /api/v1/speakers/john.doe`
   - Cross-service references MUST use meaningful IDs

2. **ADR-004** (Factor User Fields from Domain Entities):
   - Speaker stores only `username` - NO foreign key (cross-service)
   - API responses combine User + Speaker fields via HTTP enrichment
   - No duplication of email, name, bio, photo, company fields
   - User data fetched from Company User Management Service at runtime

3. **Data Models** (`03-data-architecture.md`, Section "Speaker"):
   - Speaker entity schema with arrays for expertise, topics, languages
   - GIN indexes for array search

4. **API Design** (`04-api-speaker-coordination.md`):
   - Speaker CRUD endpoints
   - Resource expansion (`?include=events,sessions,companies`)
   - Advanced search filters

### Database Schema Reference

**Architecture Schema (Source of Truth):**
- Doc: `docs/architecture/03-data-architecture.md`, Section "Speaker"
- Tables: `speakers` (new), existing `speaker_pool`

```sql
-- ADR-003/ADR-004 Compliant Schema
-- NOTE: Uses username (meaningful ID), NOT user_id UUID
-- NO foreign key constraint (cross-service reference)
CREATE TABLE speakers (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    username VARCHAR(100) NOT NULL UNIQUE,  -- ADR-003: meaningful identifier
    availability VARCHAR(50) DEFAULT 'available',
    workflow_state VARCHAR(50) DEFAULT 'identified',
    expertise_areas TEXT[],
    speaking_topics TEXT[],
    linkedin_url VARCHAR(500),
    twitter_handle VARCHAR(100),
    certifications TEXT[],
    languages VARCHAR(10)[] DEFAULT ARRAY['de', 'en'],
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
    -- NO FK to users table (ADR-004: cross-service, use HTTP enrichment)
);

CREATE UNIQUE INDEX idx_speakers_username ON speakers(username);
CREATE INDEX idx_speakers_availability ON speakers(availability);
CREATE INDEX idx_speakers_workflow_state ON speakers(workflow_state);
CREATE INDEX idx_speakers_expertise_areas ON speakers USING GIN(expertise_areas);
```

### Speaker Entity (JPA)

```java
@Entity
@Table(name = "speakers")
@Data
@Builder
@NoArgsConstructor
@AllArgsConstructor
public class Speaker {

    @Id
    @GeneratedValue(strategy = GenerationType.AUTO)
    @Column(columnDefinition = "UUID")
    private UUID id;

    /**
     * Username as meaningful identifier (ADR-003).
     * References User in Company User Management Service.
     * NO foreign key constraint (cross-service per ADR-004).
     * User data enriched via HTTP at runtime.
     */
    @Column(name = "username", nullable = false, unique = true, length = 100)
    private String username;

    @Column(name = "availability", length = 50)
    @Enumerated(EnumType.STRING)
    private SpeakerAvailability availability = SpeakerAvailability.AVAILABLE;

    @Column(name = "workflow_state", length = 50)
    @Convert(converter = SpeakerWorkflowStateConverter.class)
    private SpeakerWorkflowState workflowState = SpeakerWorkflowState.IDENTIFIED;

    @Column(name = "expertise_areas", columnDefinition = "TEXT[]")
    @Type(ListArrayType.class)
    private List<String> expertiseAreas = new ArrayList<>();

    @Column(name = "speaking_topics", columnDefinition = "TEXT[]")
    @Type(ListArrayType.class)
    private List<String> speakingTopics = new ArrayList<>();

    @Column(name = "linkedin_url", length = 500)
    private String linkedInUrl;

    @Column(name = "twitter_handle", length = 100)
    private String twitterHandle;

    @Column(name = "certifications", columnDefinition = "TEXT[]")
    @Type(ListArrayType.class)
    private List<String> certifications = new ArrayList<>();

    @Column(name = "languages", columnDefinition = "VARCHAR(10)[]")
    @Type(ListArrayType.class)
    private List<String> languages = List.of("de", "en");

    @Column(name = "created_at", nullable = false)
    private Instant createdAt;

    @Column(name = "updated_at", nullable = false)
    private Instant updatedAt;

    @PrePersist
    protected void onCreate() {
        createdAt = Instant.now();
        updatedAt = Instant.now();
    }

    @PreUpdate
    protected void onUpdate() {
        updatedAt = Instant.now();
    }
}
```

### User Enrichment Pattern (ADR-004)

```java
@Service
@RequiredArgsConstructor
public class SpeakerService {

    private final SpeakerRepository speakerRepository;
    private final UserServiceClient userServiceClient;
    private final DomainEventPublisher eventPublisher;

    public SpeakerResponse getSpeakerByUsername(String username, Set<String> includes) {
        // 1. Get Speaker profile from database (by username - ADR-003)
        Speaker speaker = speakerRepository.findByUsername(username)
            .orElseThrow(() -> new SpeakerNotFoundException(username));

        // 2. Get User from Company User Management Service (HTTP enrichment - ADR-004)
        UserDto user = userServiceClient.getUserByUsername(username);

        // 3. Combine User + Speaker fields (ADR-004 HTTP enrichment)
        SpeakerResponse response = SpeakerResponse.builder()
            // From User entity (via HTTP):
            .username(user.getUsername())
            .email(user.getEmail())
            .firstName(user.getFirstName())
            .lastName(user.getLastName())
            .bio(user.getBio())
            .profilePictureUrl(user.getProfilePictureUrl())
            .companyName(user.getCompanyId())
            // From Speaker entity (local):
            .availability(speaker.getAvailability())
            .workflowState(speaker.getWorkflowState())
            .expertiseAreas(speaker.getExpertiseAreas())
            .speakingTopics(speaker.getSpeakingTopics())
            .linkedInUrl(speaker.getLinkedInUrl())
            .twitterHandle(speaker.getTwitterHandle())
            .certifications(speaker.getCertifications())
            .languages(speaker.getLanguages())
            .build();

        // 4. Handle includes
        if (includes.contains("events")) {
            response.setEvents(fetchSpeakerEvents(username));
        }
        if (includes.contains("sessions")) {
            response.setSessions(fetchSpeakerSessions(username));
        }

        return response;
    }

    @Transactional
    public SpeakerResponse createSpeaker(CreateSpeakerRequest request) {
        // Verify user exists (ADR-004: validate via HTTP)
        UserDto user = userServiceClient.getUserByUsername(request.getUsername());

        // Store username as reference (ADR-003: meaningful identifier)
        Speaker speaker = Speaker.builder()
            .username(request.getUsername())  // NOT userId UUID
            .expertiseAreas(request.getExpertiseAreas())
            .speakingTopics(request.getSpeakingTopics())
            .linkedInUrl(request.getLinkedInUrl())
            .twitterHandle(request.getTwitterHandle())
            .languages(request.getLanguages())
            .build();

        speaker = speakerRepository.save(speaker);

        // Publish domain event
        eventPublisher.publish(new SpeakerCreatedEvent(
            speaker.getUsername(),
            speaker.getId()
        ));

        return toResponse(speaker, user);
    }
}
```
