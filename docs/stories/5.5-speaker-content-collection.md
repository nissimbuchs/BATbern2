# Story 5.5: Speaker Content Collection

## Status
Accepted

## Story

**As an** organizer,
**I want** to collect speaker content (title, abstract, CV, photo),
**so that** I can prepare event materials and publish speaker information.

## Domain Context

### Primary Domain

**Event Management Domain** - Organizer workflows and speaker content management

This story implements **Workflow Step 6** of the 16-step event workflow (FR2):
- Speaker Content Collection (title, abstract, CV, photo)
- Organizer-driven content upload (on behalf of speakers)
- Optional: Simple speaker self-submission form (no authentication required)

### Involved Services

**Services:**
- **Speaker Coordination Service**: Content collection API and business logic (controllers, services)
- **Event Management Service**: Data persistence layer (speaker_materials table, repositories)
- **API Gateway**: Routes speaker materials endpoints
- **AWS S3**: File storage for CVs and photos with presigned URL uploads

**Architecture Note**:
- **Data Layer**: `speaker_materials` table resides in **event-management-service** database (colocated with speaker_pool from Story 5.2)
- **Service Layer**: Speaker-coordination-service provides the API and business logic
- **Storage**: AWS S3 for file storage (CVs, photos), CloudFront CDN for delivery
- **Upload Pattern**: Presigned URLs for direct S3 uploads (per ADR-002 - never proxy files through backend)

**Shared-Kernel Foundation** (automatically included):

### File Upload Pattern (ADR-002)

**Presigned URL Flow**:
1. Frontend requests presigned URL: `POST /api/v1/events/{code}/speakers/{speakerId}/materials/presigned-url` with `{ fileType: 'CV' | 'PHOTO', contentType: 'application/pdf' | 'image/jpeg' }`
2. Backend generates presigned URL via AWS SDK: `s3Client.presignPutObject()` with 15-minute expiration
3. Frontend uploads directly to S3 via PUT request to presigned URL (with progress tracking)
4. Frontend calls API to save metadata: `POST /api/v1/events/{code}/speakers/{speakerId}/materials` with `{ cv_url: 's3://...', ... }`
5. Backend saves speaker_materials record with S3 URLs

**S3 Bucket Structure**:
```
batbern-speaker-materials-{env}/
  ├── cvs/
  │   └── {event_code}/
  │       └── {speaker_id}/
  │           └── cv_{timestamp}.pdf
  └── photos/
      └── {event_code}/
          └── {speaker_id}/
              └── photo_{timestamp}.jpg
```

### Workflow Integration

```java
// In SpeakerMaterialsService.submitMaterials()

@Transactional
public SpeakerMaterialsResponse submitMaterials(
    String eventCode,
    UUID speakerId,
    String organizerUsername,
    SubmitMaterialsRequest request
) {
    // 1. Validate speaker exists and is in valid state
    SpeakerPool speaker = speakerPoolRepository.findById(speakerId)
        .orElseThrow(() -> new NotFoundException("Speaker not found: " + speakerId));

    if (speaker.getStatus() != SpeakerWorkflowState.ACCEPTED) {
        throw new ValidationException("Speaker must be ACCEPTED before content collection");
    }

    // 2. Create or update speaker_materials record
    SpeakerMaterials materials = speakerMaterialsRepository
        .findBySpeakerPoolId(speakerId)
        .orElse(new SpeakerMaterials());

    materials.setSpeakerPoolId(speakerId);
    materials.setPresentationTitle(request.getTitle());
    materials.setAbstract(request.getAbstract());
    materials.setCvUrl(request.getCvUrl());
    materials.setPhotoUrl(request.getPhotoUrl());
    materials.setStatus(MaterialStatus.SUBMITTED);
    materials.setSubmissionType(SubmissionType.ORGANIZER_UPLOAD);
    materials.setSubmittedByUsername(organizerUsername);
    materials.setSubmittedAt(Instant.now());

    speakerMaterialsRepository.save(materials);

    // 3. Transition speaker workflow state (delegates to Story 5.1a workflow service)
    speakerWorkflowService.updateSpeakerWorkflowState(
        speaker.getSessionId(),
        speakerId,
        SpeakerWorkflowState.CONTENT_SUBMITTED,
        organizerUsername
    );

    // 4. Publish domain event (DomainEventPublisher from shared-kernel)
    domainEventPublisher.publishEvent(new SpeakerMaterialsSubmittedEvent(
        eventCode,
        speakerId,
        SubmissionType.ORGANIZER_UPLOAD,
        organizerUsername,
        materials.getSubmittedAt()
    ));

    return mapToResponse(materials);
}
```

## Dev Notes - Implementation Guide

### Content Completion Calculation

**Completion Percentage Logic**:
- 0%: No fields submitted
- 25%: Title only
- 50%: Title + Abstract
- 75%: Title + Abstract + CV
- 100%: Title + Abstract + CV + Photo

**Formula**: `(filledFields / 4) * 100` where filledFields counts: title, abstract, cv_url, photo_url

### S3 Presigned URL Security

**Security Considerations**:
- Presigned URLs expire after 15 minutes
- Use PUT method (not POST) to prevent CSRF attacks
- Validate file content-type on client and server
- Scan uploaded files for malware (optional with AWS Macie)
- Implement rate limiting on presigned URL generation (max 10 per minute per organizer)

### Speaker Self-Submission UX

**Public Form Requirements**:
- No authentication required (public URL)
- URL format: `/speakers/submit?eventCode={code}&speakerId={id}`
- Simple, clean UI with minimal friction
- Success page with confirmation message
- Error handling for invalid eventCode or speakerId
- Email notification to organizer on submission

### Bulk Upload Feature (AC6)

**CSV Template Format**:
```csv
speakerId,presentationTitle,abstract,cvUrl,photoUrl
123e4567-e89b-12d3-a456-426614174000,"Microservices at Scale","Lorem ipsum...",https://s3.amazonaws.com/...,https://s3.amazonaws.com/...
223e4567-e89b-12d3-a456-426614174001,"Event-Driven Design","Lorem ipsum...",https://s3.amazonaws.com/...,https://s3.amazonaws.com/...
```

**Implementation**:
- Endpoint: `POST /api/v1/events/{code}/speakers/materials/bulk` with multipart/form-data
- Parse CSV, validate each row, create speaker_materials records in transaction
- Return summary: `{ totalRows: 10, successCount: 8, errorCount: 2, errors: [...] }`
