## Story

**As an** organizer,
**I want** to send speaker invitations from the speaker management UI with a single click,
**so that** I can efficiently invite speakers without using API tools or scripts.

### API Endpoint (Existing)

```yaml
POST /api/v1/events/{eventCode}/speakers/{username}/send-invitation
  Request Body (optional):
    responseDeadline?: string  # ISO date, default: 30 days
    personalMessage?: string   # Custom message in email

  Response 200 OK:
    token: string              # Magic link token (for testing)
    workflowState: "INVITED"
    invitedAt: string          # ISO timestamp
    email: string              # Where invitation was sent

  Response 400 Bad Request:
    error: "SPEAKER_NOT_FOUND" | "ALREADY_INVITED" | "INVALID_STATE"
```

## Acceptance Criteria

### AC1: Send Invitation Button in Speaker Drawer

1. When viewing a speaker in IDENTIFIED state, show "Send Invitation" button
2. Button is primary/prominent action in the drawer
3. Button disabled for speakers not in IDENTIFIED state
4. Button shows loading state while sending

### AC2: Send Invitation API Integration

1. Clicking button calls `POST /events/{code}/speakers/{username}/send-invitation`
2. On success:
   - Toast: "Invitation sent to {email}"
   - Speaker card updates to show INVITED state
   - Drawer closes or updates to show new state
3. On failure:
   - Toast with error message
   - Button re-enabled for retry

### AC3: Kanban Card Actions

1. IDENTIFIED cards show "Invite" quick action icon (mail/send icon)
2. Icon is **disabled** (greyed out with tooltip) when speaker has no email
3. Clicking enabled icon shows **confirmation tooltip**: "Send invitation to {email}?" with Send/Cancel
4. On confirmation, send invitation with loading spinner on icon
5. Card moves to INVITED column on success
6. Show email icon badge (✉️) on INVITED cards to indicate invitation sent
7. On failure, show error tooltip on card with retry option

### AC4: Email Requirement

1. Speaker must have email to send invitation
2. In **drawer**: If no email, show inline email input field above Send button
3. In **kanban**: If no email, invite icon disabled with tooltip "Add email in details to invite"
4. Email validation: Must be valid email format (standard regex)
5. After valid email entered in drawer, Save + Send flow:
   - Save email to speaker record first
   - Then enable Send Invitation button
   - Do NOT auto-send (user must click Send)

## Dev Notes - Implementation Guide

### Service Method

```typescript
// speakerPoolService.ts
export interface SendInvitationRequest {
  responseDeadline?: string;
  personalMessage?: string;
}

export interface SendInvitationResponse {
  token: string;
  workflowState: string;
  invitedAt: string;
  email: string;
}

export const speakerPoolService = {
  // ... existing methods

  sendInvitation: async (
    eventCode: string,
    username: string,
    options?: SendInvitationRequest
  ): Promise<SendInvitationResponse> => {
    const response = await apiClient.post(
      `/events/${eventCode}/speakers/${username}/send-invitation`,
      options || {}
    );
    return response.data;
  },
};
```

### Hook

```typescript
// useSendInvitation.ts
export function useSendInvitation(eventCode: string) {
  const queryClient = useQueryClient();

  return useMutation({
    mutationFn: ({ username, options }: { username: string; options?: SendInvitationRequest }) =>
      speakerPoolService.sendInvitation(eventCode, username, options),
    onSuccess: () => {
      queryClient.invalidateQueries({ queryKey: ['speakerPool', eventCode] });
      toast.success(t('speakers.invitationSent'));
    },
    onError: (error) => {
      toast.error(t('speakers.invitationFailed'));
    },
  });
}
```

### Drawer Button

```tsx
// SpeakerOutreachDetailsDrawer.tsx
{speaker?.status === 'IDENTIFIED' && (
  <Button
    variant="contained"
    color="primary"
    startIcon={<SendIcon />}
    onClick={() => sendInvitation.mutate({ username: speaker.username })}
    disabled={sendInvitation.isPending || !speaker.email}
  >
    {sendInvitation.isPending ? t('common.sending') : t('speakers.sendInvitation')}
  </Button>
)}
```
