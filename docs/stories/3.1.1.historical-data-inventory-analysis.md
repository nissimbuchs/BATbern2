## Acceptance Criteria

1. **Data Source Catalog**: Complete inventory of all data files in `apps/BATspa-old/` with file paths, sizes, and counts
2. **JSON Schema Documentation**: Detailed schema definition for each JSON file (sessions.json, topics.json, pictures.json) with field types and sample values
3. **File Asset Inventory**: Complete list of all PDF presentations, speaker photos, partner logos, and event images with file sizes and formats
4. **Data Quality Report**: Identification of missing data, inconsistencies, duplicates, and format issues in the source data
5. **Volume Metrics**: Exact counts for:
   - Total events (BAT sessions)
   - Total sessions/presentations
   - Unique speakers
   - Unique companies
   - PDF/PPTX files
   - Image files (photos, logos)
6. **Sample Data Validation**: Manual verification of 10 randomly selected events for data correctness and completeness
7. **Data Quality Issues Documented**: List of all data quality issues found with severity (critical, warning, info) and recommended remediation

### JSON Schema Analysis

**sessions.json structure:**
```typescript
interface Session {
  bat: number;                    // Event/BAT number (1-60+)
  pdf: string;                    // PDF filename
  title: string;                  // Presentation title
  abstract?: string;              // Presentation abstract (optional)
  authoren?: string;              // Authors (for program PDFs)
  referenten?: Speaker[];         // Speakers array
}

interface Speaker {
  name: string;                   // "FirstName LastName, Company"
  bio: string;                    // Speaker biography
  company: string;                // Company identifier
  portrait: string;               // Photo filename
}
```

**topics.json structure:**
```typescript
interface Topic {
  bat: number;                    // Event/BAT number
  topic: string;                  // Event topic/theme
  datum: string;                  // Date string (various formats)
  eventType: string;              // "Abend-BAT", "Halb-BAT", "Ganztag-BAT"
}
```

**pictures.json structure:**
```typescript
interface Picture {
  bat: number;                    // Event/BAT number
  picture: string;                // Image filename
  caption?: string;               // Optional caption
}
```

### Data Quality Checks to Perform

**1. Completeness Checks:**
- Sessions without abstracts
- Sessions without speakers (authoren only = program PDFs)
- Speakers without bios
- Speakers without portraits
- Events without presentations

**2. Referential Integrity:**
- PDF files referenced but not found on disk
- Portrait images referenced but not found
- Events referenced in sessions but not in topics.json

**3. Data Consistency:**
- Duplicate speaker names with different bios
- Inconsistent company names (case, abbreviations)
- Date format variations in topics.json
- Event type inconsistencies

**4. Data Completeness:**
- Events (BAT numbers 1-60+) that have no sessions
- Missing event metadata (location, attendance)
