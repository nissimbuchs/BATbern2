## Architecture Context

### Architecture Patterns

**Referenced from architecture docs (SOURCE OF TRUTH):**

1. **ADR-003** (Meaningful Identifiers in Public APIs):
   - Invitation references Speaker via `username` (NOT UUID)
   - Response token is the public identifier for speaker self-service
   - Example: `GET /api/v1/invitations/respond/{token}`

2. **ADR-004** (Factor User Fields from Domain Entities):
   - Invitation stores only `username` - NO foreign key (cross-service)
   - Personalization data (speaker name, email) fetched from User Service at send time

3. **Speaker Workflow State Machine** (`04-api-speaker-coordination.md`):
   - `identified` → `contacted`: When invitation is sent
   - `contacted` → `accepted`/`declined`: When speaker responds via token
   - State transitions trigger domain events

4. **AWS SES Integration** (`04-api-core.md`):
   - Email templates stored with variable substitution
   - Template type: `SPEAKER_INVITATION`
   - Bounce and complaint handling integrated

### Database Schema Reference

**Architecture Schema (Source of Truth):**
- Doc: `docs/architecture/03-data-architecture.md`, Section "Notification Domain Models"
- Tables: `speaker_invitations` (new), existing `speakers`

```sql
-- ADR-003/ADR-004 Compliant Schema
-- NOTE: Uses username (meaningful ID), NOT user_id UUID
-- NO foreign key constraint (cross-service reference)
CREATE TABLE speaker_invitations (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    username VARCHAR(100) NOT NULL,           -- ADR-003: meaningful identifier
    event_code VARCHAR(50) NOT NULL,          -- ADR-003: meaningful identifier
    response_token VARCHAR(64) NOT NULL UNIQUE, -- Unique token for passwordless response
    invitation_status VARCHAR(50) NOT NULL DEFAULT 'pending',
    sent_at TIMESTAMP WITH TIME ZONE,
    responded_at TIMESTAMP WITH TIME ZONE,
    response_type VARCHAR(50),                -- 'accepted', 'declined', 'tentative'
    decline_reason TEXT,
    expires_at TIMESTAMP WITH TIME ZONE NOT NULL,
    reminder_count INTEGER DEFAULT 0,
    last_reminder_at TIMESTAMP WITH TIME ZONE,
    email_message_id VARCHAR(255),            -- AWS SES message ID for tracking
    email_opened_at TIMESTAMP WITH TIME ZONE,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    created_by VARCHAR(100) NOT NULL,         -- Organizer username who sent invitation
    -- NO FK to speakers table (cross-service, ADR-004)
    -- NO FK to events table (cross-service)
    CONSTRAINT valid_invitation_status CHECK (invitation_status IN ('pending', 'sent', 'opened', 'responded', 'expired'))
);

CREATE UNIQUE INDEX idx_speaker_invitations_token ON speaker_invitations(response_token);
CREATE INDEX idx_speaker_invitations_username ON speaker_invitations(username);
CREATE INDEX idx_speaker_invitations_event_code ON speaker_invitations(event_code);
CREATE INDEX idx_speaker_invitations_status ON speaker_invitations(invitation_status);
CREATE INDEX idx_speaker_invitations_expires_at ON speaker_invitations(expires_at) WHERE invitation_status NOT IN ('responded', 'expired');
```

            type: string

        required: true

              $ref: '#/components/schemas/RespondToInvitationRequest'

        '200':
