# Story 2.8.1: Partner Directory

## Status
Done

## Story

**As an** organizer,
**I want** a Partner Directory screen with search, filtering, and list/grid views,
**so that** I can efficiently browse and manage partner companies with visual tier indicators and quick access to partner details.

## Domain Context

### Primary Domain
Partner Analytics Domain - Partner directory interface for organizer workflows

- **Frontend**: React 19 application (TypeScript, Material-UI v5, Zustand, React Query)
- **Backend**: Partner Coordination Service (REST APIs from Story 2.7)
- **Backend**: Company Management Service (for company data enrichment via HTTP)
- **Backend**: User Management Service (for contact data enrichment via HTTP)

## Architecture Context

### Architecture Patterns
[Source: architecture/05-frontend-architecture.md]
- **Component Architecture**: Atomic design with presentational/container separation
- **State Management**:
  - Zustand for UI state (filters, view mode, search query, sort order)
  - React Query for server state (partners list, statistics, cache management)
- **Routing**: React Router v6 with lazy loading
- **Form Management**: React Hook Form with Zod validation
- **API Path Pattern**: Use `/partners` (WITHOUT `/api/v1` prefix - already in baseURL)

[Source: architecture/04-api-design.md#partners-api]
- **API Client**: Axios with interceptors for auth/error handling
- **Resource Expansion**: Uses `?include=company,contacts` to reduce API calls (ADR-004 HTTP enrichment)
- **Filtering**: Query parameter syntax: `?filter=partnershipLevel:gold,isActive:true`
- **Sorting**: Query parameter: `?sort=companyName:asc` or `?sort=partnershipLevel:desc`
- **Pagination**: Query parameters: `?page=0&size=20` (default page size: 20)
- **Caching**: React Query with 2min staleTime for partner lists, 10min for contacts

[Source: architecture/03-data-architecture.md#partner-aggregate]
- **Partner Entity**: UUID id, companyName (meaningful ID), partnershipLevel enum, dates, isActive
- **Partnership Levels**: BRONZE, SILVER, GOLD, PLATINUM, STRATEGIC (with emoji indicators)
- **TypeScript Types**: Generated from `docs/api/partners-api.openapi.yml`

## Acceptance Criteria

1. [ ] **Partner Directory Screen** displays partner list with search and filtering
   - Search by partner name (debounced 300ms)
   - Filter by partnership tier (dropdown: All, Strategic, Platinum, Gold, Silver, Bronze)
   - Filter by status (toggle: All, Active, Inactive)
   - Quick filter chips for tier selection (🏆 Strategic, 💎 Platinum, 🥇 Gold, 🥈 Silver, 🥉 Bronze)
   - Sort dropdown (Engagement Score, Name, Tier, Last Event)
   - Grid/List view toggle persisted in Zustand

2. [ ] **Partner Overview Statistics** displays aggregated metrics
   - Total partners count
   - Active partners count (isActive=true)
   - Engaged partners percentage (Epic 8 - show placeholder "Coming Soon")
   - Tier distribution breakdown (count per tier with emoji indicators)

3. [ ] **Partner Cards** display comprehensive information
   - Company logo (from CloudFront CDN, fallback to initials avatar)
   - Partnership tier badge with emoji (🏆/💎/🥇/🥈/🥉)
   - Company name (companyName from API)
   - Industry (from enriched company data via `?include=company`)
   - Primary contact name and email (from enriched contact data via `?include=contacts`)
   - Last event attendance (from partner data)
   - Topic votes count (from partner votes API)
   - Next meeting date (from partner meetings API) or "Pending"
   - Engagement bar (Epic 8 - show placeholder)
   - Quick action buttons: [View Details] [Send Email - Epic 8] [Schedule Meeting - Epic 8] [Analytics - Epic 8]

4. [ ] **Pagination** handles large partner lists
   - Default page size: 20 partners
   - Page navigation controls (first, previous, next, last)
   - Page number display ("Page 1 of 5")
   - Total count display ("24 partners")

5. [ ] **Navigation Integration** adds Partners to top navigation
   - "Partners" menu item in organizer navigation bar (same level as Events, Companies, Users)
   - Active state indicator when on Partners screens
   - Role-based visibility (organizers only)
   - Route: `/partners`

6. [ ] **Responsive Design** works on mobile, tablet, desktop
   - Mobile (<640px): Single column, stacked filters, simplified cards
   - Tablet (640-1024px): 2 columns grid, collapsible filter panel
   - Desktop (>1024px): 3 columns grid, side filter panel

7. [ ] **Accessibility** meets WCAG 2.1 AA standards
   - Keyboard navigation support (Tab, Enter, Escape)
   - Screen reader announcements for filter changes, search results
   - ARIA labels for all interactive elements
   - Focus indicators visible
   - Color contrast ratios ≥4.5:1

8. [ ] **Performance** meets targets
   - Initial page load <2s
   - Search/filter interactions <200ms
   - Image lazy loading for logos
   - React Query caching reduces API calls (2min stale time)

9. [ ] **Internationalization** supports German and English
   - All UI text uses i18n keys
   - Tier labels translated (Strategic → Strategisch)
   - Date formatting uses locale (DD.MM.YYYY for German)
   - Number formatting (engagement percentages)

10. [ ] **Integration Tests** verify API integration
    - Test partner list API call with filters
    - Test search API call with query parameter
    - Test pagination API calls
    - Test statistics API call
    - Test error handling (404, 500, network errors)
    - Test HTTP enrichment via `?include=company,contacts`

**API Client (partnerApi.ts):**
```typescript
// src/services/api/partnerApi.ts
import apiClient from './apiClient';
import type { components } from '@/types/generated/partner-api.types';

// CRITICAL: Use path WITHOUT /api/v1 prefix (already in baseURL)
const PARTNER_API_PATH = '/partners';

export type PartnerResponse = components['schemas']['PartnerResponse'];
export type PartnerListResponse = components['schemas']['PartnerListResponse'];
export type PartnerStatistics = components['schemas']['PartnerStatistics']; // Epic 8 feature
export type PartnershipLevel = components['schemas']['PartnershipLevel'];

export interface PartnerFilters {
  tier?: PartnershipLevel | 'all';
  status?: 'all' | 'active' | 'inactive';
}

export interface PartnerPagination {
  page: number;
  size: number;
}

export interface PartnerSort {
  sortBy: 'engagement' | 'name' | 'tier' | 'lastEvent';
  sortOrder: 'asc' | 'desc';
}

export const listPartners = async (
  filters: PartnerFilters,
  sort: PartnerSort,
  pagination: PartnerPagination
): Promise<PartnerListResponse> => {
  const params: Record<string, string | number> = {
    page: pagination.page,
    size: pagination.size,
    include: 'company,contacts', // ADR-004 HTTP enrichment
  };

  // Build filter query parameter
  const filterParts: string[] = [];
  if (filters.tier && filters.tier !== 'all') {
    filterParts.push(`partnershipLevel:${filters.tier}`);
  }
  if (filters.status === 'active') {
    filterParts.push('isActive:true');
  } else if (filters.status === 'inactive') {
    filterParts.push('isActive:false');
  }
  if (filterParts.length > 0) {
    params.filter = filterParts.join(',');
  }

  // Build sort query parameter
  params.sort = `${sort.sortBy}:${sort.sortOrder}`;

  const response = await apiClient.get(PARTNER_API_PATH, { params });
  // ✅ Resolves to: http://localhost:8080/api/v1/partners?page=0&size=20&include=company,contacts&filter=...&sort=...
  return response.data;
};

export const getPartnerStatistics = async (): Promise<any> => {
  // Epic 8 feature - return mock data for now
  const response = await apiClient.get(`${PARTNER_API_PATH}/statistics`);
  return response.data;
};
```

**React Query Hook (usePartners.ts):**
```typescript
// src/hooks/usePartners.ts
import { useQuery } from '@tanstack/react-query';
import { listPartners, PartnerFilters, PartnerSort, PartnerPagination } from '@/services/api/partnerApi';

export const usePartners = (
  filters: PartnerFilters,
  sort: PartnerSort,
  pagination: PartnerPagination
) => {
  return useQuery({
    queryKey: ['partners', filters, sort, pagination],
    queryFn: () => listPartners(filters, sort, pagination),
    staleTime: 2 * 60 * 1000, // 2 minutes
    cacheTime: 10 * 60 * 1000, // 10 minutes
    refetchOnWindowFocus: true,
    retry: 2,
  });
};
```

```typescript
// src/stores/partnerStore.ts
import { create } from 'zustand';
import { persist } from 'zustand/middleware';
import type { PartnershipLevel } from '@/services/api/partnerApi';

interface PartnerFilters {
  tier: PartnershipLevel | 'all';
  status: 'all' | 'active' | 'inactive';
}

interface PartnerStore {
  // State
  filters: PartnerFilters;
  viewMode: 'grid' | 'list';
  searchQuery: string;
  sortBy: 'engagement' | 'name' | 'tier' | 'lastEvent';
  sortOrder: 'asc' | 'desc';
  page: number;

  // Actions
  setFilters: (filters: Partial<PartnerFilters>) => void;
  setViewMode: (mode: 'grid' | 'list') => void;
  setSearchQuery: (query: string) => void;
  setSortBy: (sortBy: PartnerStore['sortBy']) => void;
  setSortOrder: (order: 'asc' | 'desc') => void;
  setPage: (page: number) => void;
  resetFilters: () => void;
}

export const usePartnerStore = create<PartnerStore>()(
  persist(
    (set) => ({
      // Initial state
      filters: { tier: 'all', status: 'all' },
      viewMode: 'grid',
      searchQuery: '',
      sortBy: 'engagement',
      sortOrder: 'desc',
      page: 0,

      // Actions
      setFilters: (newFilters) =>
        set((state) => ({
          filters: { ...state.filters, ...newFilters },
          page: 0, // Reset to first page on filter change
        })),
      setViewMode: (mode) => set({ viewMode: mode }),
      setSearchQuery: (query) => set({ searchQuery: query, page: 0 }),
      setSortBy: (sortBy) => set({ sortBy, page: 0 }),
      setSortOrder: (order) => set({ sortOrder: order }),
      setPage: (page) => set({ page }),
      resetFilters: () =>
        set({
          filters: { tier: 'all', status: 'all' },
          searchQuery: '',
          page: 0,
        }),
    }),
    {
      name: 'partner-store', // localStorage key
      partialState: (state) => ({ viewMode: state.viewMode }), // Only persist viewMode
    }
  )
);
```

### Technical Design Notes

**Tier Badge Mapping:**
- STRATEGIC: 🏆 Strategic
- PLATINUM: 💎 Platinum
- GOLD: 🥇 Gold
- SILVER: 🥈 Silver
- BRONZE: 🥉 Bronze

**HTTP Enrichment Pattern (ADR-004):**
- Use `?include=company,contacts` query parameter
- Backend makes HTTP calls to Company Service and User Service
- Frontend receives enriched response with company and contact data
- Reduces frontend API calls from 3 to 1

**Search Debouncing:**
- Use custom `useDebouncedValue` hook with 300ms delay
- Prevents excessive API calls while user is typing
- Clear button to reset search

**Pagination:**
- Default page size: 20
- Zero-based page indexing (page=0 for first page)
- Display "Page X of Y" with total count
- Disable first/prev on first page, next/last on last page
