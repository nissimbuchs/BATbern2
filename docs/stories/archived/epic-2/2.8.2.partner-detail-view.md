# Story 2.8.2: Partner Detail View

## Status
Done

## Story

**As an** organizer,
**I want** a Partner Detail screen with tabbed interface showing comprehensive partner information,
**so that** I can view partnership details, contacts, meetings, activity history, and notes in an organized layout with quick access to partner actions.

## Domain Context

### Primary Domain
Partner Analytics Domain - Partner detail interface for organizer workflows

### Involved Services
- **Frontend**: React 19 application (TypeScript, Material-UI v5, Zustand, React Query)
- **Backend**: Partner Coordination Service (REST APIs from Story 2.7)
- **Backend**: Company Management Service (for company data enrichment via HTTP)
- **Backend**: User Management Service (for contact data enrichment via HTTP)
- **CDN**: CloudFront for company logo delivery
- **Cache**: React Query for client-side caching (5min TTL for detail, 2min for activity)

**Shared-Kernel Foundation** (automatically included):
- ErrorResponse (standardized error format with correlationId, path, severity)
- HTTP client utilities (JWT propagation, error handling)

**OpenAPI Code Generation**:
- Generate TypeScript types from `docs/api/partners-api.openapi.yml`
- Import shared types via openapi-typescript v7.10.1

## Architecture Context

### Architecture Patterns
[Source: architecture/05-frontend-architecture.md]
- **Component Architecture**: Atomic design with presentational/container separation
- **State Management**:
  - Zustand for UI state (active tab, modal visibility)
  - React Query for server state (partner detail, contacts, votes, meetings, activity)
- **Routing**: React Router v6 with dynamic route parameter `:companyName`
- **Tabs**: Material-UI Tabs component with lazy loading for tab panels
- **API Path Pattern**: Use `/partners/{companyName}` (WITHOUT `/api/v1` prefix)

[Source: architecture/04-api-design.md#partners-api]
- **API Client**: Axios with interceptors for auth/error handling
- **Resource Expansion**: Uses `?include=company,contacts,votes,meetings,activity` to reduce API calls
- **Caching**: React Query with 5min staleTime for detail, 2min for activity timeline

[Source: architecture/03-data-architecture.md#partner-aggregate]
- **Partner Entity**: UUID id, companyName (meaningful ID - ADR-003), partnershipLevel, dates, isActive
- **PartnerContact**: username (meaningful ID), contactRole, isPrimary
- **TopicVote**: topicId, voteValue (1-5), voteWeight, votedAt
- **PartnerMeeting**: meetingType, scheduledDate, location, agenda

### Database Schema Reference
N/A - Frontend story, no database migrations required. Backend schema handled in Story 2.7.

### Infrastructure Components
[Source: architecture/02-infrastructure-deployment.md]

**Frontend Deployment:**
- **S3 + CloudFront**: Static React app hosting
- **Environment Config**: Runtime environment variables via window.__ENV__
- **API Base URL**: Configured per environment (local: http://localhost:8080/api/v1)

**CDN Integration:**
- **Company Logos**: Served via CloudFront with 1-year max-age
- **Assets**: Vite build with content hashing for cache busting

**Routing:**
- Dynamic route: `/partners/:companyName` (e.g., `/partners/GoogleZH`)
- Route parameter validation (companyName max 12 chars)

## Acceptance Criteria

1. [ ] **Partner Detail Header** displays comprehensive partner information
   - Company logo (CloudFront CDN, fallback to initials avatar)
   - Partnership tier badge with emoji (🏆 STRATEGIC, 💎 PLATINUM, 🥇 GOLD, 🥈 SILVER, 🥉 BRONZE)
   - Company name (companyName from API)
   - Industry, website, location (from enriched company data via `?include=company`)
   - Engagement bar (placeholder "Coming Soon - Epic 8")
   - Action buttons: [📧 Send Email - Epic 8], [📅 Schedule Meeting - Epic 8], [📊 View Analytics - Epic 8], [📝 Add Note], [Edit Partner]
   - [← Back to Partner Directory] button

2. [ ] **Quick Stats Cards** display key metrics
   - Partner Since: date and duration (e.g., "January 2022 (3 years)")
   - Events Attended: count and last event (e.g., "24 (last: Spring 25)")
   - Active Votes: count (e.g., "5 (topics)")
   - Meetings: count (e.g., "12 (seasonal)")

3. [ ] **Tab Navigation** provides organized content access
   - Tab labels: Overview, Contacts, Meetings, Activity, Notes, Settings
   - Active tab indicator (underline, color highlight)
   - Keyboard navigation (Left/Right arrows, Home, End)
   - Lazy loading for tab panels (only load data when tab activated)
   - Active tab persisted in URL hash (e.g., `/partners/GoogleZH#contacts`)

4. [ ] **Overview Tab** displays partnership and activity details
   - **Partnership Details Panel**:
     - Partnership Tier with emoji badge
     - Tier Start Date (e.g., "Jan 1, 2024")
     - Previous Tier (e.g., "💎 Platinum")
     - Benefits list (✓ Logo placement, ✓ Newsletter, ✓ Priority access, etc.)
     - Swiss UID and Tax Status (from company data)
     - [Change Tier] button (opens modal - Story 2.8.3)
   - **Engagement Metrics Panel** (placeholder):
     - "Coming Soon - Epic 8" message
     - Score breakdown placeholders (Event Attendance, Topic Voting, Meeting Participation, Content Interaction)
     - [View Full Analytics →] link (Epic 8)
   - **Recent Activity Panel**:
     - Last event attendance (date, attendee count, registrations, downloads)
     - Active topic votes list (topic name, priority: High/Medium/Low)
     - Next meeting preview

5. [ ] **Meetings Tab** displays meeting history and upcoming meetings (read-only)
   - List of meetings (type, date, location, agenda)
   - RSVP status display (read-only)
   - Meeting materials links (if available)
   - "Full meeting coordination coming in Epic 8" message
   - [+ Schedule New Meeting] button (placeholder for Epic 8)

6. [ ] **Activity Tab** displays timeline of partner activities
   - Chronological activity list (most recent first)
   - Activity types: Vote Cast, Meeting Attended, Contact Added, Tier Changed, etc.
   - Activity details (date, user, description)
   - Filter by activity type (dropdown)
   - Pagination (20 items per page)

7. [ ] **Notes Tab** provides organizer notes management
   - List of notes (title, author, date, preview)
   - [+ Add Note] button
   - Edit/Delete actions on notes (organizer-only)
   - Rich text editor for note content
   - Notes tagged with author username and timestamp

8. [ ] **Settings Tab** provides partner configuration (organizer-only)
   - Partnership status (Active/Inactive toggle)
   - Auto-renewal settings (boolean, renewal date)
   - Access permissions (Epic 8 feature)
   - [Export Partner Data] button (GDPR compliance - Epic 8)
   - [Delete Partner Data] button (GDPR compliance - Epic 8, requires confirmation)

9. [ ] **Responsive Design** works on mobile, tablet, desktop
   - Mobile (<640px): Single column, vertical tabs, stacked panels
   - Tablet (640-1024px): Two columns for side-by-side panels, horizontal tabs
   - Desktop (>1024px): Three columns, horizontal tabs, full panels

10. [ ] **Accessibility** meets WCAG 2.1 AA standards
    - Keyboard navigation for tabs (Arrow keys, Home, End)
    - Screen reader announcements for tab changes, content updates
    - ARIA labels for all interactive elements
    - Focus indicators visible
    - Color contrast ratios ≥4.5:1

11. [ ] **Performance** meets targets
    - Initial page load <2s
    - Tab switching <200ms
    - Lazy load tab content (only fetch data when tab activated)
    - Image lazy loading for company logo
    - React Query caching reduces API calls (5min for detail, 2min for activity)

12. [ ] **Internationalization** supports German and English
    - All UI text uses i18n keys
    - Tab labels translated
    - Date formatting uses locale
    - Partnership tier labels translated

13. [ ] **Integration Tests** verify API integration
    - Test partner detail API call with `?include=company,contacts,votes,meetings,activity`
    - Test contacts API call (Story 2.8.4)
    - Test votes API call
    - Test meetings API call
    - Test activity API call
    - Test notes API call (CRUD operations)
    - Test error handling (404 for invalid companyName, 500, network errors)

export const getPartnerDetail = async (
  companyName: string,

  const response = await apiClient.get(`${PARTNER_API_PATH}/${companyName}/votes`);
  return response.data;

export const getPartnerMeetings = async (companyName: string): Promise<MeetingResponse[]> => {
  const response = await apiClient.get(`${PARTNER_API_PATH}/${companyName}/meetings`);
  return response.data;

export const getPartnerActivity = async (
  companyName: string,
  filters?: { type?: string }
): Promise<ActivityResponse[]> => {
  const params: Record<string, string> = {};
  if (filters?.type) {
    params.filter = `type:${filters.type}`;
  }
  const response = await apiClient.get(`${PARTNER_API_PATH}/${companyName}/activity`, { params });
  return response.data;
};

export const createPartnerNote = async (
  companyName: string,

### React Query Hooks

**usePartnerDetail.ts:**
```typescript
import { useQuery } from '@tanstack/react-query';
import { getPartnerDetail } from '@/services/api/partnerApi';

export const usePartnerDetail = (companyName: string, include?: string) => {
  return useQuery({
    queryKey: ['partner', companyName, include],
    queryFn: () => getPartnerDetail(companyName, include),
    staleTime: 5 * 60 * 1000, // 5 minutes
    enabled: !!companyName,
  });
};
```

import { create } from 'zustand';

interface PartnerDetailStore {

### Technical Design Notes
