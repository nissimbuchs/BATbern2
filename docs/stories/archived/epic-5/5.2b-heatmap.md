# Story 5.2b: Multi-Topic Heat Map Visualization

## Status
✅ Implemented (Frontend Complete)

## Story

**As an** organizer,
**I want** to visualize topic usage patterns across multiple topics and BATbern events (Spring, Summer, Autumn),
**so that** I can quickly identify stale topics, trending topics, and make informed decisions about topic selection.

## Context

This story extends **Story 5.2 (Topic Selection & Speaker Brainstorming)** with an enhanced visualization mode based on the wireframe specification in `docs/wireframes/story-2.2-topic-backlog-management.md`.

### Background

The original implementation showed topics in a list view only. The wireframe specified three view modes:
- **Heat Map** - Visual representation of topic usage over time (multi-topic)
- **List** - Standard list view (existing implementation)
- **Board** - Kanban-style board view (future implementation)

Additionally, there was a single-topic heat map component (`TopicHeatMap`) that showed usage history for an individual selected topic, but no multi-topic overview.

### BATbern Event Schedule

BATbern hosts **3 events per year** (not quarterly):
- **Spring Event**: March-May
- **Summer Event**: June-September
- **Autumn Event**: October-February

## Implementation

### Frontend Components Created

#### 1. MultiTopicHeatMap Component
**File**: `/web-frontend/src/components/TopicHeatMap/MultiTopicHeatMap.tsx`

**Purpose**: Display multiple topics with their usage history across BATbern's 3 annual events over multiple years.

**Features**:
- **Multi-topic rows**: Each topic gets its own row showing usage across time
- **3 events per year**: Spring (S), Summer (S), Autumn (A) columns
- **6 years of history**: Configurable via `yearsToShow` prop (default: 6)
- **Color-coded intensity**:
  - Gray: No usage
  - Blue (light): Low usage (< 33% intensity)
  - Orange (medium): Medium usage (33-66% intensity)
  - Red (dark): High usage (> 66% intensity)
- **Status indicators**:
  - ⚠️ **Stale**: Topic not used in last 4 events (~16 months)
  - ✨ **Trending**: Topic used in last 2 events
- **Interactive**:
  - Click any topic row to view details in right panel
  - Hover over cells to see usage details (event count, attendance)
- **Legend**: Explains colors and indicators

**Props**:
```typescript
interface MultiTopicHeatMapProps {
  topics: TopicWithHistory[];           // Topics with optional usage history
  selectedTopicId?: string;             // Currently selected topic ID
  onTopicSelect: (topic: Topic) => void; // Handler when topic clicked
  yearsToShow?: number;                 // Number of years to display (default: 6)
}

interface TopicWithHistory extends Topic {
  usageHistory?: TopicUsageHistory[];
}
```

**Season Mapping Logic**:
```typescript
// Map month to BATbern season
if (month >= 2 && month <= 4) {
  season = 'spring';   // March-May
} else if (month >= 5 && month <= 8) {
  season = 'summer';   // June-September
} else {
  season = 'autumn';   // October-February
}
```

**Intensity Calculation**:
- Intensity is calculated per topic (relative to that topic's max usage)
- Formula: `(usageCount / maxUsageInTopic) * 100`
- This ensures each topic's usage pattern is visible, even if some topics are used more frequently overall

**Staleness Detection**:
- **Stale**: `stalenessScore < 30` OR no usage in last 4 events
- **Trending**: Used in last 2 events from current period

### Type Definitions

**Existing Types Used**:
- `Topic` - Base topic interface from `@/types/topic.types`
- `TopicUsageHistory` - Usage history record interface

**New Types**:
```typescript
// MultiTopicHeatMap.tsx
interface TopicWithHistory extends Topic {
  usageHistory?: TopicUsageHistory[];
}

interface EventPeriod {
  year: number;
  season: 'spring' | 'summer' | 'autumn';
  label: string; // e.g., "Spring 2024"
  key: string;   // e.g., "2024-spring"
}

interface TopicUsageCell {
  period: EventPeriod;
  usageCount: number;
  intensity: number; // 0-100
  eventIds: string[];
  totalAttendance: number;
}

interface TopicHeatMapRow {
  topic: Topic;
  cells: Map<string, TopicUsageCell>;
  totalUsage: number;
  lastUsedPeriod?: EventPeriod;
  isStale: boolean;
  isTrending: boolean;
}
```

### Interactions

1. **Switch to Heat Map**: Click "Heat Map" toggle button
2. **View Topic Details**: Click on any topic row → details panel updates
3. **Hover for Details**: Hover over usage cells to see:
   - Event period (e.g., "Spring 2024")
   - Number of events
   - Total attendance
4. **Identify Patterns**:
   - **Stale topics** (⚠️): Good candidates for reuse
   - **Trending topics** (✨): Recently used, avoid duplication
   - **Color intensity**: Darker = more frequent usage in that period

## Technical Details

### Component Architecture

```
TopicBacklogManager
├── ViewModeToggle (ToggleButtonGroup)
├── TopicFilterPanel (left sidebar)
├── Content Area (center) - Conditional:
│   ├── MultiTopicHeatMap (viewMode === 'heatMap')
│   ├── TopicList (viewMode === 'list')
│   └── BoardPlaceholder (viewMode === 'board')
└── TopicDetailsPanel (right sidebar)
```

### Data Flow

1. `useTopics(filters)` fetches topics with filters
2. Response includes topics (may or may not have `usageHistory`)
3. `MultiTopicHeatMap` receives `topics` array
4. Component processes usage history into heat map cells
5. User clicks topic → `onTopicSelect(topic)` → details panel updates

### Performance Considerations

- **Memo hooks**: `useMemo` for expensive calculations (period generation, heat map row processing)
- **Efficient rendering**: Only selected view mode renders (not all 3 views)
- **Tooltip optimization**: Tooltips render on-demand (hover)
- **Map-based lookups**: `Map<string, TopicUsageCell>` for O(1) cell lookups

### Browser Compatibility

- Uses Material-UI components (wide browser support)
- CSS Grid for layout (modern browsers)
- No external charting library dependencies for heat map (pure MUI + CSS)

## Conclusion

The multi-topic heat map visualization provides organizers with an intuitive, visual way to understand topic usage patterns across BATbern's Spring, Summer, and Autumn events over multiple years. This enhances the topic selection workflow by making it easier to:

1. **Identify safe topics** - Stale topics that haven't been used recently
2. **Avoid duplication** - Trending topics used in recent events
3. **Spot patterns** - Topics that are used seasonally or annually
4. **Make informed decisions** - Visual overview complements detailed metrics

The implementation is frontend-only, requires no backend changes, and seamlessly integrates with the existing Topic Backlog Manager from Story 5.2.
