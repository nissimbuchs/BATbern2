# Story 5.5: Speaker Content Submission, Quality Review & Configurable Task System

## Status
Done

## Story

**As an** organizer,
**I want to** submit speaker materials, have them quality-reviewed, and manage configurable event tasks,
**So that** I can coordinate all aspects of event planning with proper workflow tracking and task delegation.

---

## Context & Motivation

### Workflow Redesign Discovery (2025-12-19)

During implementation of [Stories 5.1-5.4](../prd/epic-5-enhanced-organizer-workflows.md#phase-a-event-setup-stories-51-51a-52), we discovered that the original "16-step linear workflow" was a conceptual misconception. See the complete workflow redesign analysis in [`docs/architecture/06a-workflow-state-machines.md#workflow-architecture-redesign-2025-12-19`](../architecture/06a-workflow-state-machines.md#workflow-architecture-redesign-2025-12-19).

The actual system architecture revealed:

1. **Event Workflow**: High-level states for event lifecycle (9 states, not 16 steps)
2. **Speaker Workflow**: Per-speaker state machine with parallel progression
3. **Task System**: Assignable work items (newsletters, catering, etc.) are NOT workflow states

**Key Insights:**
- Quality review and slot assignment can happen in any order per speaker
- As soon as the first speaker accepts, event progresses (not waiting for all speakers)
- Tasks like newsletters, catering, partner meetings need due dates and assignees, not workflow states

This story consolidates what were previously separate stories (5.5-5.7) into a cohesive implementation reflecting the parallel workflow reality.

## Scope

This story implements:

### A. Speaker/User Lookup & Management
- User autocomplete search from users-service
- Create new users with SPEAKER role
- Edit existing speaker profiles
- Store username as FK to users-service (not user_id)

### B. Speaker Content Submission
- Presentation title/abstract entry
- Create session + session_users junction link
- Portrait upload to S3 (presigned URLs)
- Update speaker_pool.status = 'content_submitted'

### C. Quality Review (Per Speaker)
- Review queue for submitted content
- Approve/reject with feedback
- Check abstract for lessons learned, no product promotion
- Update speaker_pool.status = 'quality_reviewed'

### D. Parallel Workflow Support
- quality_reviewed and slot_assigned can happen in any order
- Auto-update to 'confirmed' when both complete
- Visual indicators for workflow progress

### E. Configurable Task System
- Task templates (default + custom)
- Auto-create tasks on workflow state transitions
- Task dashboard for organizers
- Task completion tracking

- AND portrait is loaded if available
- AND username is stored for linking

**AC3: Create New User**
- GIVEN search finds no matching user

- THEN a modal opens with fields: name (required), email (required), company, bio, portrait upload
- AND role is auto-set to SPEAKER
- WHEN saved, user is created in users-service
- THEN username is returned and used for linking

**AC4: Edit Existing Speaker**
- GIVEN a speaker is already linked
- WHEN organizer clicks "Edit speaker profile"
- THEN a modal opens with current user data
- WHEN changes are saved
- THEN users-service is updated
- AND changes are visible across all sessions for this speaker

**AC5: Username Storage**
- GIVEN a user is selected or created
- THEN username is stored in speaker_pool (assigned_organizer_id pattern)
- AND username is stored in session_users junction table
- AND no user_id is stored (deprecated FK)

### B. Content Submission (AC6-10)

**AC6: Content Form**
- GIVEN an organizer is submitting content for an accepted speaker
- WHEN they open the content submission form
- THEN they see fields: presentation title (required), abstract (required, max 1000 chars), portrait upload
- AND character counter shows remaining characters for abstract

**AC7: Session Creation**
- GIVEN organizer submits content with valid title and abstract

- THEN a new session record is created in `sessions` table

- GIVEN session is created

- THEN a session_users record is created
- WITH session_id = created session
- AND username = speaker username
- AND speaker_role = 'primary_speaker'
- AND is_confirmed = false (until both quality_reviewed AND slot_assigned)

**AC9: Portrait Upload**
- GIVEN organizer uploads speaker portrait

**AC16: Flexible Order**

### F. Error Handling & Edge Cases (AC28-37)

**AC28: Users-Service Unavailability**
- GIVEN users-service is unavailable
- WHEN organizer attempts to create new speaker
- THEN system retries 3 times with exponential backoff (1s, 2s, 4s)
- AND displays "Users service temporarily unavailable. Please try again." error message
- AND does NOT create speaker_pool entry (transaction rolled back)
- AND organizer can retry manually

**AC29: Ambiguous User Search Results**
- GIVEN multiple users have similar names
- WHEN organizer searches for a name
- THEN all matching users are displayed with name, email, company
- AND users with SPEAKER role are highlighted with badge
- AND organizer can distinguish users by email/company
- AND selection is based on unique username (not name)

**AC30: Duplicate User Creation Prevention**
- GIVEN organizer attempts to create speaker with email that already exists
- WHEN user creation is submitted
- THEN system checks users-service for existing user with same email
- AND if found, uses existing user's username (no duplicate created)
- AND if not found, creates new user
- AND organizer is notified which path was taken

**AC31: Portrait Upload URL Generation Failure**
- GIVEN AWS S3 is unavailable or credentials are invalid
- WHEN organizer requests portrait upload URL
- THEN system logs error with details
- AND displays "Unable to generate upload URL. Please contact support." to user
- AND portrait upload is marked as optional (can proceed without)
- AND organizer can retry upload later via "Edit speaker profile" (AC4)

**AC32: Portrait Upload Partial Failure Recovery**
- GIVEN portrait uploads to S3 successfully
- WHEN users-service update fails
- THEN system logs orphaned S3 object key for cleanup
- AND displays warning: "Portrait uploaded but not linked to profile. Our team will fix this shortly."
- AND S3 lifecycle policy deletes temp uploads after 7 days
- AND organizer can re-upload portrait later (overwrites orphaned object)

**AC33: Content Submission Transaction Integrity**
- GIVEN content submission involves multiple database writes (session, session_users, speaker_pool)
- WHEN any database write fails
- THEN entire transaction is rolled back (no partial state)
- AND no orphaned session records exist
- AND no orphaned session_users records exist
- AND speaker_pool.status remains unchanged
- AND organizer sees error: "Content submission failed. Please try again."
- AND organizer can retry submission without data corruption

**AC34: Session Deletion Protection**
- GIVEN speaker_pool references a session via session_id FK
- WHEN session is deleted (accidentally or intentionally)
- THEN speaker_pool.session_id is set to NULL (not cascade deleted)
- AND speaker_pool record is preserved
- AND speaker status is reset to 'accepted' if it was content_submitted/quality_reviewed/confirmed
- AND organizer is warned "Content was lost. Please resubmit." when viewing speaker
- AND organizer can resubmit content to create new session

**AC35: Concurrent Workflow State Update Handling**
- GIVEN two organizers simultaneously update speaker state (one approves quality, one assigns slot)
- WHEN both trigger checkAndUpdateToConfirmed() concurrently
- THEN optimistic locking detects concurrent modification
- AND losing thread retries with fresh data
- AND speaker is updated to 'confirmed' exactly once
- AND no race condition leaves speaker in inconsistent state
- AND both organizers see success message

**AC36: Task Auto-Creation Idempotency**
- GIVEN event transitions to a state that triggers task creation
- WHEN workflow transition event is published multiple times (retry/duplicate)
- THEN tasks are created only once per template per event
- AND unique constraint prevents duplicate task insertion
- AND duplicate event is logged and skipped
- AND no error is thrown to user

**AC37: Content Submission State Validation**
- GIVEN speaker is in status other than 'accepted'
- WHEN organizer attempts to submit content
- THEN system rejects with error: "Speaker must be accepted before content submission"
- AND no session is created
- AND speaker status remains unchanged
- AND organizer is directed to change speaker status to 'accepted' first

**SpeakerContentSubmissionService.java** - Multi-step transaction with external service call

**Story-Specific Implementation:**
- External call to users-service (getOrCreateUser) runs in separate transaction with REQUIRES_NEW propagation
- Main transaction covers: session creation, session_users link, speaker_pool update, event publishing
- Transaction isolation: READ_COMMITTED, timeout: 10 seconds, rollback on all exceptions (AC33)
- Validates speaker status='accepted' before allowing content submission (AC37)

**QualityReviewService.java** - Approval workflow with parallel state management

**Story-Specific Implementation:**
- `approveContent()` updates speaker to 'quality_reviewed' and calls `checkAndUpdateToConfirmed()`
- `checkAndUpdateToConfirmed()` implements parallel workflow (AC16-AC17): If quality_reviewed AND session.start_time exists → auto-update to 'confirmed'
- `rejectContent()` preserves status='content_submitted' and stores feedback in speaker_pool.notes (AC14)
- Updates both speaker_pool.status AND session_users.is_confirmed for consistency (AC17)

**EventTaskService.java** - Event-driven task auto-creation with idempotency

**Story-Specific Implementation:**
- Implements `ApplicationListener<EventWorkflowTransitionEvent>` to auto-create tasks on workflow transitions
- Idempotency: Batch check via `existsByEventIdAndTriggerState()` + per-task check + unique DB constraint (AC36)
- Due date calculation: immediate, relative_to_event (offset days), or absolute (AC19)
- Template matching: Only creates tasks for templates with matching trigger_state (AC23)

## Error Handling & Resilience

### External Service Failures

**Users-Service Integration (AC28-AC30):**
```java
@Service
public class UserApiClient {

    @Retryable(
        value = {HttpServerErrorException.class, ResourceAccessException.class},
        maxAttempts = 3,
        backoff = @Backoff(delay = 1000, multiplier = 2) // 1s, 2s, 4s
    )
    public UserDto createUser(CreateUserRequest request) {
        try {
            return restTemplate.postForObject(
                usersServiceUrl + "/api/users",
                request,
                UserDto.class
            );
        } catch (HttpClientErrorException.ServiceUnavailable e) {
            throw new ExternalServiceException(
                "Users service temporarily unavailable. Please try again in a moment.",
                "USERS_SERVICE_UNAVAILABLE",
                true // retryable
            );
        } catch (SocketTimeoutException e) {
            throw new ExternalServiceException(
                "Request to users service timed out. Please try again.",
                "USERS_SERVICE_TIMEOUT",
                true
            );
        }
    }

    // Email uniqueness check for duplicate prevention (AC30)
    public String getOrCreateSpeakerUsername(CreateUserRequest request) {
        Optional<UserDto> existing = findByEmail(request.getEmail());

        if (existing.isPresent()) {
            log.info("User with email {} already exists, using existing username: {}",
                     request.getEmail(), existing.get().getUsername());
            return existing.get().getUsername();
        }

        UserDto created = createUser(request);
        return created.getUsername();
    }
}
```

**Configuration:**
- **Retry policy**: 3 attempts with exponential backoff (1s, 2s, 4s)
- **Timeout**: 5 seconds per request
- **Circuit breaker**: Open after 5 consecutive failures, half-open after 30s
- **Fallback**: Display clear error message to user, allow manual retry

**AWS S3 Integration (AC31-AC32):**
```java
@Service
public class PortraitUploadService {

    public PresignedUrlResponse requestPortraitUpload(String username) {
        try {
            String objectKey = String.format("portraits/temp/%s-%s.jpg",
                username, UUID.randomUUID());

            PresignedPutObjectRequest presignedRequest = PresignedPutObjectRequest.builder()
                .signatureDuration(Duration.ofMinutes(15))
                .putObjectRequest(r -> r
                    .bucket(s3Config.getPortraitBucket())
                    .key(objectKey)
                    .contentType("image/jpeg"))
                .build();

            PresignedPutObjectUrl presignedUrl = s3Presigner.presignPutObject(presignedRequest);

            return PresignedUrlResponse.builder()
                .uploadUrl(presignedUrl.url().toString())
                .expiresAt(Instant.now().plus(Duration.ofMinutes(15)))
                .objectKey(objectKey)
                .build();

        } catch (S3Exception e) {
            log.error("Failed to generate presigned URL for user {}: {}", username, e.getMessage(), e);
            throw new StorageException(
                "Unable to generate upload URL. Please try again or contact support.",
                "S3_PRESIGN_FAILED",
                false // not retryable by user
            );
        }
    }
}
```

### Transaction Boundaries (AC33)

**Content Submission Transaction:**
```java
@Service
public class SpeakerContentSubmissionService {

    @Transactional(
        isolation = Isolation.READ_COMMITTED,
        rollbackFor = Exception.class,
        timeout = 10 // seconds
    )
    public SpeakerContentDto submitContent(String eventId, String poolId,
                                          SubmitContentRequest request) {
        try {
            // 1. Validate speaker_pool entry
            SpeakerPool speaker = speakerPoolRepository.findById(poolId)
                .orElseThrow(() -> new EntityNotFoundException("Speaker not found"));

            if (!speaker.getStatus().equals("accepted")) {
                throw new InvalidWorkflowStateException(
                    "Speaker must be accepted before content submission"
                );
            }

            // 2. Get/create user (external call, NOT in transaction)
            String username = getOrCreateSpeakerUsername(request);

            // 3. Create session
            Session session = sessionRepository.save(Session.builder()
                .eventId(UUID.fromString(eventId))
                .title(request.getPresentationTitle())
                .description(request.getPresentationAbstract())
                .sessionType("presentation")
                .build());

            // 4. Create session_users link
            sessionUserRepository.save(SessionUser.builder()
                .sessionId(session.getId())
                .username(username)
                .speakerRole("primary_speaker")
                .isConfirmed(false)
                .build());

            // 5. Update speaker_pool
            speaker.setSessionId(session.getId());
            speaker.setStatus("content_submitted");
            speakerPoolRepository.save(speaker);

            // 6. Publish event (only if all above succeeded)
            eventPublisher.publishEvent(new SpeakerWorkflowStateChangeEvent(
                poolId, eventId, "accepted", "content_submitted", username
            ));

            return toDto(speaker, session);

        } catch (DataIntegrityViolationException e) {
            log.error("Data integrity violation during content submission for pool {}: {}",
                     poolId, e.getMessage(), e);
            // Transaction will auto-rollback due to @Transactional
            throw new ContentSubmissionException(
                "Unable to submit content due to data inconsistency. Please contact support.",
                "DATA_INTEGRITY_VIOLATION",
                e
            );
        } catch (Exception e) {
            log.error("Unexpected error during content submission for pool {}: {}",
                     poolId, e.getMessage(), e);
            // Transaction will auto-rollback
            throw new ContentSubmissionException(
                "Content submission failed. Please try again.",
                "SUBMISSION_FAILED",
                e
            );
        }
    }
}
```

**Transaction Scope:**
- **Includes**: Session creation, session_users link, speaker_pool update, event publication
- **Excludes**: Users-service calls (external system), S3 presigned URL generation (idempotent), email notifications (async, best-effort)
- **Rollback**: All database changes rolled back on any exception
- **Timeout**: 10 seconds maximum transaction duration

### Concurrency Control (AC35)

**Optimistic Locking:**
```java
@Entity
@Table(name = "speaker_pool")
public class SpeakerPool {

    @Id
    private UUID id;

    @Version // JPA Optimistic Locking
    private Long version;

    private String status;
    private UUID sessionId;
    // ... other fields
}

@Service
public class SpeakerWorkflowService {

    @Transactional
    public void checkAndUpdateToConfirmed(SpeakerPool speaker) {
        try {
            boolean isQualityReviewed = "quality_reviewed".equals(speaker.getStatus());
            boolean hasSlotAssigned = speaker.getSessionId() != null &&
                sessionRepository.findById(speaker.getSessionId())
                    .map(session -> session.getStartTime() != null)
                    .orElse(false);

            if ((isQualityReviewed && hasSlotAssigned) ||
                ("slot_assigned".equals(speaker.getStatus()) && isContentQualityReviewed(speaker))) {

                speaker.setStatus("confirmed");
                speakerPoolRepository.save(speaker); // Optimistic lock check happens here

                updateSessionUserConfirmation(speaker, true);
                eventPublisher.publishEvent(new SpeakerConfirmedEvent(/*...*/));
            }

        } catch (OptimisticLockException e) {
            log.info("Concurrent update detected for speaker {}. Retrying confirmation check.",
                     speaker.getId());

            // Retry with fresh data
            SpeakerPool freshSpeaker = speakerPoolRepository.findById(speaker.getId())
                .orElseThrow();
            checkAndUpdateToConfirmed(freshSpeaker);
        }
    }
}
```

**Optimistic Locking Strategy:**
- `speaker_pool.version` column (JPA @Version annotation)
- `event_tasks.version` column (JPA @Version annotation)
- Retry on OptimisticLockException with fresh data (max 3 attempts)
- No pessimistic locking required (low contention expected)

### Idempotency (AC36)

**Task Auto-Creation:**
```java
@Service
public class EventTaskService implements ApplicationListener<EventWorkflowTransitionEvent> {

    @Override
    @Transactional
    public void onApplicationEvent(EventWorkflowTransitionEvent event) {
        String triggeredState = event.getNewState().name().toLowerCase();
        String eventId = event.getEventId();

        // Idempotency check: Have we already processed this transition?
        boolean alreadyProcessed = eventTaskRepository.existsByEventIdAndTriggerState(
            UUID.fromString(eventId),
            triggeredState
        );

        if (alreadyProcessed) {
            log.info("Tasks already created for event {} state {}. Skipping duplicate processing.",
                     eventId, triggeredState);
            return;
        }

        List<TaskTemplate> templates = templateRepository.findByTriggerState(triggeredState);

        for (TaskTemplate template : templates) {
            createTaskFromTemplate(eventId, template, event.getEvent().getEventDate());
        }
    }

    private void createTaskFromTemplate(String eventId, TaskTemplate template,
                                       LocalDateTime eventDate) {
        // Additional idempotency check at task level
        boolean taskExists = eventTaskRepository.existsByEventIdAndTemplateId(
            UUID.fromString(eventId),
            template.getId()
        );

        if (taskExists) {
            log.debug("Task {} already exists for event {}. Skipping.",
                     template.getName(), eventId);
            return;
        }

        EventTask task = EventTask.builder()
            .eventId(UUID.fromString(eventId))
            .templateId(template.getId())
            .taskName(template.getName())
            .triggerState(template.getTriggerState())
            .dueDate(calculateDueDate(template, eventDate))
            .assignedOrganizerUsername(template.getDefaultAssignee())
            .status("todo")
            .build();

        eventTaskRepository.save(task);
    }
}
```

**Idempotency Mechanisms:**
- Check `existsByEventIdAndTriggerState()` before creating tasks
- Unique index on `(event_id, template_id)` prevents duplicates at DB level
- Log and skip duplicate events (no error thrown)
- Safe to replay workflow transition events

### Data Integrity (AC34)

**Session Deletion Protection:**
```java
@Service
public class SpeakerContentService {

    public SpeakerContentDto getSpeakerContent(String poolId) {
        SpeakerPool speaker = speakerPoolRepository.findById(poolId)
            .orElseThrow(() -> new EntityNotFoundException("Speaker not found"));

        if (speaker.getSessionId() == null) {
            return SpeakerContentDto.builder()
                .poolId(speaker.getId())
                .status(speaker.getStatus())
                .hasContent(false)
                .build();
        }

        Optional<Session> session = sessionRepository.findById(speaker.getSessionId());

        if (session.isEmpty()) {
            // Session was deleted - data inconsistency detected
            log.warn("Speaker {} references deleted session {}. Unlinking and resetting status.",
                     poolId, speaker.getSessionId());

            // Fix inconsistency
            speaker.setSessionId(null);
            if ("content_submitted".equals(speaker.getStatus()) ||
                "quality_reviewed".equals(speaker.getStatus()) ||
                "confirmed".equals(speaker.getStatus())) {
                speaker.setStatus("accepted");
            }
            speakerPoolRepository.save(speaker);

            return SpeakerContentDto.builder()
                .poolId(speaker.getId())
                .status(speaker.getStatus())
                .hasContent(false)
                .warning("Content was lost. Please resubmit.")
                .build();
        }

        return toDto(speaker, session.get());
    }
}
```

**Database Constraint:**
- FK constraint: `ON DELETE SET NULL` (not CASCADE DELETE)
- Preserves speaker_pool record when session deleted
- Application code detects and recovers from orphaned FK
- User notified to resubmit content

## Notes & Decisions

**2025-12-19:** Story redesigned to consolidate content submission, quality review, and task system based on workflow architecture discovery during Stories 5.1-5.4 implementation. Key insight: parallel speaker workflows vs linear event workflow.

**2025-12-20:** Added comprehensive error handling and edge case coverage (AC28-AC37) based on checklist validation. Key additions:
- External service failure handling (users-service, S3)
- Transaction boundary clarifications
- Optimistic locking for concurrent updates
- Idempotency for task auto-creation
- Data integrity protection for session deletion
- Detailed resilience patterns and recovery mechanisms

**Username vs User_ID:** Using username as FK to users-service instead of user_id (deprecated). This aligns with API-based user data access pattern (see V8 migration).

**No CV Upload:** Based on user feedback, CV upload is not required. Only portrait, bio, company, email stored in users-service.

**Task System Design:** Tasks are separate from workflow states to allow flexible due dates, assignments, and customization without affecting core workflow logic.

**Error Handling Philosophy:** Fail gracefully with clear user feedback, retry automatically for transient failures (network), allow manual retry for recoverable errors, and preserve data integrity above all else.

**Schema Redundancy (Technical Debt):** Three tables track overlapping speaker/session data:
- `speaker_pool`: Workflow states (identified → confirmed), speaker metadata (name, company), session_id FK
- `sessions`: Presentation content (title, description), time slot data (start_time, end_time, room)
- `session_users`: Junction table with username, speaker_role, presentation_title (REDUNDANT with sessions.title), is_confirmed (REDUNDANT with speaker_pool.status)

**Current Issues:**
1. Presentation title stored in TWO places: `sessions.title` AND `session_users.presentation_title`
2. Confirmed state stored in TWO places: `speaker_pool.status='confirmed'` AND `session_users.is_confirmed=true`
3. Speaker-to-session link via TWO mechanisms: `speaker_pool.session_id` AND `session_users(session_id, username)`

**For Story 5.5:** Using existing V20 schema (speaker_pool.session_id) and maintaining consistency by updating BOTH speaker_pool.status AND session_users.is_confirmed when speaker reaches confirmed state. Future refactoring needed to consolidate schema (see architectural debt tracking).
