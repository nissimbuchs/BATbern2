## Status
Done

## ⚡ ADR-006 Simplification

**Unified user profile approach reduces implementation effort by ~50% vs dual-table (ADR-005)**:
- ❌ **NO** anonymous_* fields in Registration entity (7 fields removed)
- ❌ **NO** account linking logic in registrations (claimed_by, claimed_at removed)
- ✅ **Simple**: attendee_id always references user_profiles (anonymous users have cognito_user_id = NULL)
- ✅ **Cleaner**: Account linking is 3 lines of code (UPDATE user_profiles.cognito_user_id)
- ✅ **Better**: Company as FK to companies table (not free text)

**Estimated Effort**: 2-3 hours (vs 4-6 hours for ADR-005)

## Story
**As a** backend developer,
**I want** to implement anonymous event registration support in the Event Management Service,
**so that** the frontend can support both authenticated and anonymous registration flows per ADR-006 (unified user profile approach).

## Domain Context

### Primary Domain
Attendee Experience Domain (Backend support for registration)

### Involved Services
- Event Management Service (Java/Spring Boot)
- PostgreSQL (registrations table)
- API Gateway (public endpoint security configuration)

### Cross-Domain Dependencies
- **Depends on**: Story 4.1.5a (Migrations V9 + V11, ADR-006, OpenAPI spec)
- **Integrates with**: Company-User-Management Service (user_profiles for account creation and linking)
- **Security Pattern**: Public endpoints using confirmation code as access token
- **Key ADR-006 Benefit**: 85% code reduction vs dual-table (ADR-005 rejected)

## Acceptance Criteria

1. **Migrations Applied**: V9 (registrations) + V11 (user_profiles with nullable cognito_user_id)
2. **User Profile Creation**: POST `/api/v1/events/{eventCode}/registrations` creates user_profile with cognito_user_id = NULL for anonymous users
3. **Confirmation Code Format**: Generated in `BAT-YYYY-NNNNNN` format using PostgreSQL sequence
4. **Public Retrieval**: GET `/api/v1/events/{eventCode}/registrations/{registrationCode}` works without authentication, returns user data from user_profiles
5. **Public Update**: PATCH `/api/v1/events/{eventCode}/registrations/{registrationCode}` updates preferences without authentication
6. **Public Cancellation**: DELETE `/api/v1/events/{eventCode}/registrations/{registrationCode}` cancels registration without authentication
7. **Company FK Integrity**: Registration request references company_id (FK to companies table), not free text
8. **Unified User Reference**: registrations.attendee_id always NOT NULL (references user_profiles)
9. **Security Configuration**: Anonymous endpoints use `permitAll()`, admin endpoints remain `authenticated()`
10. **Component Tests Pass**: All registration flows tested with real PostgreSQL (Testcontainers)
11. **Account Linking Works**: When anonymous user creates Cognito account, user_profile.cognito_user_id updated (3 lines of code)

## Dev Notes - Implementation Guide

### Confirmation Code Generation

**Database Sequence (already in V9 migration):**
```sql
CREATE SEQUENCE IF NOT EXISTS registration_code_seq START 1;
```

**Entity @PrePersist Method:**
```java
@PrePersist
protected void onCreate() {
    if (this.registrationCode == null) {
        // Generate confirmation code: BAT-YYYY-NNNNNN
        int year = LocalDate.now().getYear();
        Long sequenceValue = // fetch NEXTVAL('registration_code_seq') via native query
        this.registrationCode = String.format("BAT-%d-%06d", year, sequenceValue);
    }
    this.createdAt = Instant.now();
    this.updatedAt = Instant.now();
}
```

**Alternative (simpler):** Let database DEFAULT generate confirmation code, then read it back after INSERT.

### Mutual Exclusivity Validation

**Database Level (V9 migration):**
```sql
ADD CONSTRAINT check_registration_type
  CHECK (
    (attendee_id IS NOT NULL AND anonymous_email IS NULL) OR
    (attendee_id IS NULL AND anonymous_email IS NOT NULL)
  );
```

**Application Level (optional additional validation):**
```java
@PrePersist
@PreUpdate
protected void validateRegistrationType() {
    boolean hasAttendee = attendeeId != null;
    boolean hasAnonymous = anonymousEmail != null;
    if (hasAttendee && hasAnonymous) {
        throw new IllegalStateException("Registration must be EITHER authenticated OR anonymous, not both");
    }
    if (!hasAttendee && !hasAnonymous) {
        throw new IllegalStateException("Registration must have either attendeeId or anonymousEmail");
    }
}
```

### Security Configuration Example

**SecurityConfig.java:**
```java
@Bean
public SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {
    http
        .csrf(csrf -> csrf.disable())

            .requestMatchers(HttpMethod.POST, "/api/v1/events/*/registrations").permitAll()
            .requestMatchers(HttpMethod.GET, "/api/v1/events/*/registrations/*").permitAll()
            .requestMatchers(HttpMethod.PATCH, "/api/v1/events/*/registrations/*").permitAll()
            .requestMatchers(HttpMethod.DELETE, "/api/v1/events/*/registrations/*").permitAll()
            .requestMatchers(HttpMethod.GET, "/api/v1/events/*/registrations/*/qr").permitAll()

            // Admin endpoints (authentication required)
            .requestMatchers(HttpMethod.GET, "/api/v1/events/*/registrations").authenticated()

            // All other requests require authentication
            .anyRequest().authenticated()
        );
    return http.build();
}
```

### QR Code Generation (Story 4.1.6 dependency)

**Add ZXing dependency to build.gradle:**
```gradle
dependencies {
    implementation 'com.google.zxing:core:3.5.1'
    implementation 'com.google.zxing:javase:3.5.1'
}
```

**QRCodeService.java:**
```java
@Service
public class QRCodeService {
    public byte[] generateQRCode(String data, int width, int height) throws IOException {
        QRCodeWriter qrCodeWriter = new QRCodeWriter();
        BitMatrix bitMatrix = qrCodeWriter.encode(data, BarcodeFormat.QR_CODE, width, height);

        ByteArrayOutputStream pngOutputStream = new ByteArrayOutputStream();
        MatrixToImageWriter.writeToStream(bitMatrix, "PNG", pngOutputStream);
        return pngOutputStream.toByteArray();
    }
}
```

**Controller Method:**
```java
@GetMapping("/registrations/{registrationCode}/qr")
public ResponseEntity<byte[]> getRegistrationQR(
    @PathVariable String eventCode,
    @PathVariable String registrationCode,
    @RequestParam(defaultValue = "300") int size
) throws IOException {
    Registration registration = registrationService.getRegistrationByConfirmationCode(eventCode, registrationCode);

    // Generate QR code with registration data (JSON format)
    String qrData = String.format(
        "{\"registrationCode\":\"%s\",\"eventCode\":\"%s\"}",
        registrationCode, eventCode
    );

    byte[] qrCodeImage = qrCodeService.generateQRCode(qrData, size, size);

    HttpHeaders headers = new HttpHeaders();
    headers.setContentType(MediaType.IMAGE_PNG);
    headers.setContentLength(qrCodeImage.length);

    return ResponseEntity.ok()
        .headers(headers)
        .body(qrCodeImage);
}
```
