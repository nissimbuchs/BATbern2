## Maintenance

### Regenerating Screenshots

Screenshots are automatically captured by running the E2E test suite:

```bash
cd web-frontend

# Delete old screenshots
rm -rf ../docs/user-guide/assets/screenshots/workflow/phase-*/

# Run full test suite
./run-phases-a-b-c-d-e-autologin.sh

# Or with UI mode to watch
./run-phases-a-b-c-d-e-autologin.sh --ui
```

### Screenshot Naming Convention

Format: `{phase}-{sequence}-{description}.png`

- **phase**: Single letter or letter+number (a, b, b5, c, d, e)
- **sequence**: Zero-padded 2-digit number (01, 02, 03, ...)
- **description**: Kebab-case description of the screen state

Examples:
- `a-01-event-dashboard.png`
- `b5-04-content-submission-drawer-1-opened.png`
- `e-08-archived-badge-visible.png`

### When to Update

Screenshots should be regenerated when:
- UI components are redesigned
- Workflow steps are added/removed/modified
- Before major releases
- Quarterly for freshness (recommended)

### Version Control

All screenshots are tracked in git to ensure documentation matches the codebase version.
